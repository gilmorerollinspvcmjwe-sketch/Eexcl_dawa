# 祖玛模块需求规格

## Why
在当前 Excel 风格游戏中心内，高还原交付经典 Zuma（轨道彩球链 + 炮台插入消除）体验，提供与现有 PvZ/Snake/Tetris 模块形成节奏互补的"瞄准-插入-消除-连锁"玩法，满足玩家对短局高压、可复盘可重试的需求。

## What Changes
- 新增 Sheet14（祖玛主玩法）：选关、模式切换、战斗主界面、HUD、局内结算
- 新增 Sheet15（祖玛图鉴与练习实验）：球种图鉴、道具说明、轨道练习、命中率训练、回缩连锁教学
- 新增祖玛核心运行时：轨道系统、彩球链系统、炮台系统、碰撞判定、消除与回缩连锁、道具球体系
- 新增祖玛关卡数据层：8 个内容包、60+ 关卡脚本、难度曲线参数表
- 新增祖玛存档系统：关卡进度、最佳分数、最佳连锁、命中率记录

## Impact
- Affected specs: 新增独立模块，不影响现有模块
- Affected code: 新增 `src/features/zuma/` 目录、`src/components/zuma/` 目录、`src/styles/zuma.css`
- Sheet 结构: Sheet14/Sheet15 新增，Sheet1 Hub 新增祖玛入口卡

## ADDED Requirements

### Requirement: 核心玩法机制

#### 炮台与输入模型
- 系统应支持鼠标/键盘控制炮台旋转瞄准
- 系统应支持左键/空格发射当前球
- 系统应支持右键/Tab切换当前球与下一球
- 系统应支持暂停(P键)、倍速切换、重开功能
- 系统应在发射后显示命中/空枪/擦边的不同音画反馈
- 系统应保证空枪不静默失败，必须有视觉或音效提示

#### 轨道与彩球链系统
- 系统应使用离散采样样条(polyline/spline)定义轨道路径
- 系统应以 `distanceAlongTrack` 表示每颗球在轨道上的位置
- 系统应维护球链结构：chainId、balls数组、headDistance、speedFactor、isRewinding
- 系统应支持基础推进速度 + 难度倍率 + 事件修正（减速/加速）
- 系统应支持多轨或分岔轨道同时推进

#### 碰撞、插入与消除判定
- 系统应在发射球与链段碰撞时计算最近合法插入点（前插/后插）
- 系统应避免穿透与错位插入
- 系统应在插入后立即进行局部匹配扫描，从插入点向两侧扩展统计同色连续长度
- 系统应在同色连续 >= 3 时触发清除并记录得分
- 系统应在消除产生间隙后触发回缩检查
- 系统应仅在"前方无推进阻断"且"间隙存在"时执行回缩
- 系统应在回缩接合后再次触发消除扫描，允许多段连锁

#### 胜利/失败条件
- 系统应在清空所有在场球链并完成关卡目标时判定胜利
- 系统应在球链触及终点线（骷髅口/终点门）时判定失败
- 系统应显示失败线并给出危险分层预警：
  - safe: > 35% 距离
  - warning: 15%~35% 距离
  - critical: < 15% 距离
- 系统应在进入 critical 时同步升级 HUD、音效和轨道视觉反馈

#### Scenario: 祖玛核心玩法闭环
- **WHEN** 玩家进入 Sheet14 并选择关卡
- **THEN** 系统加载轨道、初始球链、颜色池和道具规则
- **WHEN** 玩家旋转炮台瞄准并发射
- **THEN** 系统判定碰撞位置并执行插入
- **WHEN** 插入后形成 >= 3 同色连续
- **THEN** 系统触发消除、回缩和连锁判定
- **WHEN** 球链触及终点线
- **THEN** 系统判定失败并显示结算面板
- **WHEN** 清空所有球链
- **THEN** 系统判定胜利并显示结算面板

### Requirement: 游戏界面元素

#### HUD布局（战斗中）
- 系统应在顶部两行内显示主要信息，避免压缩主战场
- 系统应显示：关卡编号/模式、当前分数、连锁层数、最高连锁
- 系统应显示：当前球 + 下一球（带颜色预览）
- 系统应显示：轨道推进百分比与危险等级
- 系统应提供：暂停、倍速、重开按钮
- 系统应使用胶囊化显示关键指标，危险状态用轻量高对比色标

#### 棋盘/战场布局
- 系统应渲染轨道几何（弯道、交叉、遮挡）
- 系统应渲染彩球链（颜色、道具球标记）
- 系统应渲染炮台（当前球、下一球、瞄准方向）
- 系统应渲染终点线（骷髅口/终点门）
- 系统应渲染道具球特效（爆裂范围、闪电方向等）

#### 结算面板
- 系统应在胜利时显示：用时、命中率、最高连锁、道具使用效率、奖励星级
- 系统应在失败时显示：失守轨道、最后20秒关键事件、推荐训练项
- 系统应提供：重开本局、返回选关、进入练习三个出口

#### Excel风格要求
- 系统应保持工作簿风格：公式栏显示当前状态摘要
- 系统应保持状态栏显示连锁/奖励提示
- 系统应保持Sheet标签切换功能

#### Scenario: HUD信息完整性
- **WHEN** 玩家进入战斗状态
- **THEN** 系统在顶部显示关卡、分数、连锁、当前球/下一球
- **WHEN** 球链进入危险区域
- **THEN** 系统升级危险等级显示并触发预警音效
- **WHEN** 触发连锁消除
- **THEN** 系统在状态栏显示连锁层数和奖励倍率

### Requirement: 关卡设计规则

#### 难度梯度（4段）
- 系统应支持入门段（1~12关）：2~3色、单轨、低速，重点学插入与换弹
- 系统应支持进阶段（13~28关）：4色、弯道/遮挡、首次道具球，加入回缩连锁教学
- 系统应支持高压段（29~45关）：多轨或分岔、速度波动、连续危机窗口
- 系统应支持终局段（46~60关）：高色数、复杂轨道、Boss规则或多目标通关

#### 关卡目标类型
- 系统应支持清空球链目标
- 系统应支持限定发射数目标
- 系统应支持限定时间目标
- 系统应支持连锁次数目标
- 系统应支持道具使用目标

#### 关卡脚本结构
- 系统应定义每关：id、trackId、colorPool、baseSpeed、spawnScript
- 系统应定义每关：powerupPool、winCondition、lossCondition、recommendedPracticeTags
- 系统应遵循"渐进扩色"原则，不在前期一次性上满颜色池

#### 节奏控制要点
- 系统应在每关前20%时间给策略建立空间
- 系统应在中段进入稳定高压
- 系统应在末段提供"收官冲刺"
- 系统应采用"波段式压力"而非全程线性加速

#### Scenario: 关卡难度递进
- **WHEN** 玩家完成入门段关卡
- **THEN** 系统解锁进阶段关卡并增加颜色种类
- **WHEN** 玩家进入高压段关卡
- **THEN** 系统启用多轨推进和速度波动
- **WHEN** 玩家进入终局段关卡
- **THEN** 系统启用Boss规则或高色数挑战

### Requirement: 角色/道具系统

#### 道具球体系（首发5类）
- 系统应支持爆裂球：命中后半径范围清除
- 系统应支持闪电球：沿轨道方向清除一段连续球
- 系统应支持减速球：全链减速N秒
- 系统应支持倒退球：链头后退固定距离，制造喘息窗口
- 系统应支持万能球：可当任意颜色参与三消

#### 道具球规则约束
- 系统应通过关卡控制道具触发率，避免纯靠道具通关
- 系统应在关卡简介显示本关道具池
- 系统应在道具球命中时显示特效和得分

#### 彩球属性
- 系统应定义彩球颜色（首发6色）
- 系统应定义彩球位置（distanceAlongTrack）
- 系统应定义彩球状态（普通/道具/待消）
- 系统应定义彩球碰撞半径

#### Scenario: 道具球触发效果
- **WHEN** 玩家命中爆裂球
- **THEN** 系统清除半径范围内所有球
- **WHEN** 玩家命中闪电球
- **THEN** 系统沿轨道方向清除一段连续球
- **WHEN** 玩家命中减速球
- **THEN** 系统全链减速指定秒数

### Requirement: 视觉风格要求

#### 色彩方案
- 系统应使用高对比度彩球颜色，便于玩家快速识别
- 系统应使用危险等级色标：safe(绿)、warning(黄)、critical(红)
- 系统应保持Excel工作簿风格底色
- 系统应使用道具球独立色系或标记

#### 动画效果
- 系统应提供球链推进动画（平滑移动）
- 系统应提供插入动画（球嵌入链段）
- 系统应提供消除动画（球消失+得分飘字）
- 系统应提供回缩动画（链段接合）
- 系统应提供连锁特效（递增视觉强度）
- 系统应提供道具球触发特效（爆裂/闪电/减速等）
- 系统应提供危险预警动画（终点线闪烁）

#### Excel风格视觉
- 系统应保持单元格栅格语义
- 系统应保持行列标识可见
- 系统应保持公式栏和状态栏风格一致

#### Scenario: 视觉反馈完整性
- **WHEN** 球链进入危险区域
- **THEN** 系统触发终点线闪烁和预警色标
- **WHEN** 触发连锁消除
- **THEN** 系统显示递增强度的连锁特效
- **WHEN** 道具球触发
- **THEN** 系统显示对应特效动画

### Requirement: 音效设计规范

#### 背景音乐
- 系统应提供关卡背景音乐（可开关）
- 系统应在危险等级升级时改变音乐节奏或强度
- 系统应在Boss关提供专属音乐

#### 操作反馈音
- 系统应提供发射音效
- 系统应提供命中音效（区分普通命中/道具命中）
- 系统应提供空枪音效（不同于命中）
- 系统应提供消除音效
- 系统应提供连锁音效（递增音调）
- 系统应提供换弹音效
- 系统应提供危险预警音效
- 系统应提供胜利/失败结算音效

#### 音效控制
- 系统应支持音效开关
- 系统应支持音量调节
- 系统应避免音效重叠冲突

#### Scenario: 音效反馈完整性
- **WHEN** 玩家发射球
- **THEN** 系统播放发射音效
- **WHEN** 球命中链段
- **THEN** 系统播放命中音效
- **WHEN** 触发连锁消除
- **THEN** 系统播放递增音调的连锁音效
- **WHEN** 球链进入危险区域
- **THEN** 系统播放预警音效

### Requirement: 性能优化指标

#### 帧率要求
- 系统应保持目标60fps稳定运行
- 系统应在常规关卡同屏球体 < 220时保持流畅
- 系统应在峰值球体 < 320时保持流畅

#### 资源占用限制
- 系统应控制单关卡内存占用 < 50MB
- 系统应控制动画渲染不阻塞主线程
- 系统应使用事件队列播放动画，避免逻辑帧阻塞

#### 调试支持
- 系统应提供调试开关：轨道采样点显示、球链索引显示、插入点候选显示、碰撞半径显示
- 系统应禁止把调试逻辑混入正式渲染路径

#### Scenario: 性能达标验证
- **WHEN** 常规关卡运行时同屏球体达到200
- **THEN** 系统保持60fps帧率
- **WHEN** 高压关卡运行时同屏球体达到300
- **THEN** 系统保持不低于45fps帧率
- **WHEN** 触发连锁消除
- **THEN** 系统动画不阻塞逻辑帧

### Requirement: 内容包体系

#### 8个正式内容包
1. 神庙征途包：经典单轨入门（12关）
2. 丛林弯道包：遮挡与提前量训练（8关）
3. 连锁回缩包：爽点强化章节（8关）
4. 机关道具包：道具球策略章节（10关）
5. 分岔多轨包：多线危机管理（8关）
6. Boss图腾包：阶段战与机制关（6关）
7. 计时冲分包：高密度短局（3档时长+6张轮换地图）
8. 无尽生存包：长期回流内容（3张无尽底图）

#### 内容包规则
- 系统应支持内容包独立接入，无需改核心状态机
- 系统应支持内容包增量上线
- 系统应首发优先交付：神庙征途包 + 连锁回缩包 + 计时冲分包

#### Scenario: 内容包接入
- **WHEN** 新增内容包数据
- **THEN** 系统自动接入关卡目录
- **WHEN** 玩家选择内容包
- **THEN** 系统加载对应关卡脚本和规则

### Requirement: 技术架构边界

#### 架构分层
- 系统应分离Simulation层与Render/UI层
- Simulation层应包含：zumaTypes.ts、zumaBoardState.ts、zumaLevelCatalog.ts、zumaProgressStorage.ts
- Render/UI层应包含：ZumaGameSheet.tsx、ZumaBoard.tsx、ZumaHud.tsx、ZumaResultPanel.tsx、zuma.css

#### 状态推进
- 系统应使用Tick驱动：updateZumaBoard(state, input, dtMs)
- 系统应使用事件流：SHOT_FIRED、BALL_INSERTED、MATCH_CLEARED、CHAIN_REWIND、POWERUP_TRIGGERED、LEVEL_WON/LEVEL_LOST
- 系统应使用事件用于HUD文案、音效触发和测试断言

#### 测试边界
- 系统应测试：插入点稳定性、三消与连锁正确性、回缩触发边界、失败线判定、道具球触发效果
- 系统应提供测试文件：zumaInsertion.test.ts、zumaMatchAndRewind.test.ts、zumaLossCondition.test.ts、zumaPowerups.test.ts

#### Scenario: 架构分层验证
- **WHEN** 运行祖玛模块
- **THEN** Simulation层与Render层保持分离
- **WHEN** 触发游戏事件
- **THEN** 事件流正确传递到HUD和音效系统
