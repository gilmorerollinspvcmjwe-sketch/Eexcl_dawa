# 推币机（Coin Pusher）Excel 游戏详细设计

- 日期：2026-04-14
- 项目：`excel-aim-trainer`
- 设计目标：在现有 Excel 风格游戏中心中交付高还原的街机推币机体验，保留"投币-推板-掉落-奖励"的核心循环，并适配 Excel 伪装风格
- 设计阶段：实施前详细设计（可直接拆解任务）

---

## 1. 游戏概述

### 1.1 一句话介绍

推币机是一款"投币入局、推板挤压、边缘掉落得分"的经典街机游戏，玩家通过控制投币时机和位置，利用推板的往复运动将硬币推下平台，获取分数与奖励。

### 1.2 核心卖点

- **即时反馈**：每次投币都能看到硬币在推板作用下移动，视觉满足感强
- **策略深度**：投币位置与时机影响整局走向，不是纯运气游戏
- **奖励驱动**：特殊币、道具卡、大奖系统形成持续追求目标
- **Excel 伪装适配**：棋盘可包装为"数据表"，硬币包装为"数据单元格"，推板包装为"滚动条"，完美融入工作簿壳层

## 2. 原版游戏分析

### 2.1 街机推币机物理机制

街机推币机的核心物理结构：

| 组件 | 功能 | 物理特性 |
|------|------|----------|
| **投币口** | 玩家投入硬币的入口 | 可左右移动，决定硬币落点 |
| **上层平台** | 硬币首次落到的区域 | 硬币在此堆积，受重力影响向前滑动 |
| **推板（Pusher）** | 前后往复运动的金属板 | 匀速或变速往复，将硬币向前推 |
| **下层平台** | 推板前方的延伸区域 | 硬币被推到此区域后可能从边缘掉落 |
| **掉落边缘** | 平台前侧和两侧的开口 | 硬币掉落即得分，前侧得分高，两侧得分低 |
| **特殊槽位** | 平台两侧的收集槽 | 硬币掉入触发特殊奖励（道具、大奖等） |

**核心物理原理**：
- 硬币之间通过接触力传递推力（推板推 A，A 推 B，B 推 C）
- 摩擦力决定硬币是否随推板一起运动
- 重力使硬币在倾斜面上自然向前滑动
- 硬币堆叠高度影响稳定性，过高时容易侧翻掉落

### 2.2 核心玩法循环

```
投币 → 硬币落在上层平台 → 推板向前推 → 硬币群整体前移
  → 边缘硬币掉落得分 → 获得新硬币/奖励 → 继续投币
```

循环节奏：
1. **决策阶段**：选择投币位置（左/中/右）和时机（推板前推/后退回）
2. **执行阶段**：投币，硬币落下并参与物理互动
3. **观察阶段**：观察推板运动对硬币群的影响
4. **收获阶段**：掉落硬币计分，获得新硬币或奖励
5. **循环**：用获得的硬币继续投币

### 2.3 奖励系统设计

原版推币机的奖励层级：

| 奖励类型 | 触发方式 | 奖励内容 |
|----------|----------|----------|
| **普通掉落** | 硬币从前侧边缘掉落 | 1:1 返还硬币 |
| **侧边掉落** | 硬币从左右两侧掉落 | 少量额外硬币或积分 |
| **特殊槽位** | 硬币掉入两侧收集槽 | 道具卡、倍率加成、小游戏 |
| **大奖触发** | 集齐特定条件（如 777） | 大量硬币倾泻（Jackpot） |
| **连锁反应** | 大量硬币同时掉落 | 额外倍率奖励 |

典型街机推币机 UI：

## 3. 核心玩法规则

### 3.1 投币规则

- **投币位置**：玩家可控制硬币落在上层平台的 X 轴位置（左/中/右三档或连续位置）
- **投币时机**：推板运动周期中，选择前推或后退回时投币，影响硬币初始受力
- **投币冷却**：每次投币后有短暂冷却（约 0.3 秒），防止连续快速投币
- **硬币库存**：玩家拥有有限数量的可用硬币，通过掉落得分补充

### 3.2 推板运动

- **运动模式**：推板沿 Y 轴做往复运动（前推 → 停顿 → 后退回 → 停顿 → 循环）
- **运动周期**：一个完整周期约 2~4 秒（随模式/难度变化）
- **推板行程**：前推距离占平台长度的 30%~50%
- **速度曲线**：前推加速 → 匀速 → 减速停顿 → 后退回加速 → 匀速 → 减速停顿

### 3.3 掉落判定

| 掉落区域 | 判定条件 | 得分倍率 |
|----------|----------|----------|
| **前侧边缘** | 硬币 Y 坐标超出平台前边界 | 1x（基础返还） |
| **左侧边缘** | 硬币 X 坐标超出平台左边界 | 0.5x + 进入左槽位概率 |
| **右侧边缘** | 硬币 X 坐标超出平台右边界 | 0.5x + 进入右槽位概率 |
| **左槽位** | 硬币落入左侧收集槽 | 触发左槽位奖励 |
| **右槽位** | 硬币落入右侧收集槽 | 触发右槽位奖励 |

### 3.4 得分计算

```
单次得分 = 掉落硬币数 × 基础分值 × 当前倍率 × 连锁加成

基础分值 = 1（每枚硬币）
当前倍率 = 1（默认），可通过道具/奖励提升
连锁加成 = 1 + (连续掉落波数 - 1) × 0.2（每多一波 +20%）
```

**得分示例**：
- 普通掉落 3 枚：3 × 1 × 1 × 1 = 3 分
- 连锁第 2 波掉落 5 枚：5 × 1 × 1 × 1.2 = 6 分
- 2 倍率下掉落 4 枚：4 × 1 × 2 × 1 = 8 分

## 4. 物理系统设计

### 4.1 简化物理模型

考虑到浏览器性能和 Excel DOM 渲染限制，采用**离散网格 + 简化物理**方案：

**核心假设**：
- 棋盘划分为离散网格（建议 12 列 × 16 行）
- 每个网格最多容纳 1 枚硬币
- 硬币运动按帧更新，非连续物理模拟
- 碰撞检测基于网格邻接关系

**物理参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 网格尺寸 | 12 × 16 | 棋盘分辨率 |
| 硬币半径 | 0.45 格 | 略小于网格，留间隙 |
| 重力系数 | 0.02 格/帧 | 自然下滑速度 |
| 摩擦系数 | 0.85 | 推板退回时硬币的跟随衰减 |
| 推力系数 | 1.0 | 推板前推时硬币的跟随程度 |
| 碰撞弹性 | 0.3 | 硬币间碰撞的能量损失 |

### 4.2 推板运动实现

```ts
// 推板状态
type PusherState = {
  position: number;       // 当前 Y 位置（0~1，0=最后，1=最前）
  phase: 'pushing' | 'paused_front' | 'returning' | 'paused_back';
  speed: number;          // 当前速度
  cycleTime: number;      // 当前周期已用时间（ms）
};

// 推板运动曲线（一个周期）
const PUSHER_CYCLE_MS = 3000;  // 3 秒一个完整周期
const PUSH_DURATION = 0.35;    // 前推占 35%
const PAUSE_FRONT = 0.10;      // 前停顿占 10%
const RETURN_DURATION = 0.35;  // 后退回占 35%
const PAUSE_BACK = 0.20;       // 后停顿占 20%
```

**推板对硬币的作用力**：
- 推板前推时：推板前方 1 格内的硬币获得推力，向前移动
- 推力传递：被推动的硬币推动其前方的硬币（链式传递，最多传递 3 层）
- 推板后退回时：硬币受摩擦力影响，有 15% 概率随推板后退 1 格

### 4.3 摩擦力模型

```
硬币是否随推板后退 = random() < (1 - frictionCoefficient)
                   = random() < 0.15

硬币自然下滑（重力）：
  每帧检查硬币前方是否为空
  若为空且硬币在倾斜区域，则向前移动
  下滑概率 = gravityCoefficient × 倾斜系数
```

### 4.4 碰撞检测

**硬币-硬币碰撞**：
- 检测 8 邻域内是否有其他硬币
- 若推板推动方向上有硬币，则传递推力
- 碰撞后两枚硬币不会重叠（网格约束）

**硬币-边界碰撞**：
- 硬币到达平台边缘时，检查是否满足掉落条件
- 掉落条件：硬币中心超出边界线 + 随机判定（模拟真实物理的不确定性）
- 掉落概率随推板推力增加而提高

**硬币-槽位碰撞**：
- 槽位位于平台左右两侧边缘
- 硬币从侧边掉落时，有概率落入槽位而非直接得分
- 槽位捕获概率 = 基础概率 + 槽位内已有硬币数 × 加成

### 4.5 物理更新循环

```
每帧（建议 30Hz 逻辑步）：
1. 更新推板位置
2. 对推板前方硬币施加推力
3. 传递推力（最多 3 层）
4. 应用重力（自然下滑）
5. 应用摩擦力（推板后退时的跟随）
6. 检测碰撞与掉落
7. 处理掉落事件（计分、奖励）
8. 更新棋盘状态
```

### 5.4 大奖系统

**Jackpot 触发条件**：
- 在限定时间内（60 秒）收集 3 枚相同的大奖标记币
- 大奖标记币仅从侧边槽位掉落，概率 5%
- 集齐 3 枚后触发 Jackpot

**Jackpot 效果**：
- 屏幕上方倾泻大量硬币（30~50 枚）
- 所有掉落硬币 ×3 倍率
- 持续 15 秒
- 视觉特效全屏庆祝

**大奖等级**：

| 等级 | 触发条件 | 倾泻硬币数 | 倍率 |
|------|----------|------------|------|
| 小奖 | 集齐 3 枚铜标记 | 20 枚 | ×2 |
| 中奖 | 集齐 3 枚银标记 | 35 枚 | ×3 |
| 大奖 | 集齐 3 枚金标记 | 50 枚 | ×5 |

## 6. 关卡/模式设计

### 6.1 经典模式（Classic）

- **定位**：完整还原街机推币机体验，默认模式
- **规则**：
  - 初始 50 枚硬币
  - 推板标准速度（3 秒/周期）
  - 标准奖励概率
  - 无时间限制，硬币用完即结束
- **目标**：尽可能获得高分
- **结算指标**：总分、最高连锁、特殊币收集数、游戏时长
- **建议关卡数**：3 个难度档位（简单/普通/困难），主要调整初始硬币数和推板速度

### 6.2 限时模式（Time Attack）

- **定位**：在限定时间内最大化得分，适合快速对局
- **规则**：
  - 初始 30 枚硬币
  - 时间限制：60 秒 / 120 秒 / 180 秒 三档
  - 推板速度略快（2.5 秒/周期）
  - 特殊币出现率 +50%
  - 时间结束时结算
- **目标**：在时间耗尽前获得最高分
- **结算指标**：总分、每秒得分效率、时间结束时剩余硬币数
- **建议关卡数**：3 个时长档 × 3 个难度 = 9 关

### 6.3 挑战模式（Challenge）

- **定位**：特殊规则下的高难度挑战，面向熟练玩家
- **规则变体**：

| 挑战名称 | 特殊规则 | 目标 |
|----------|----------|------|
| **一币通关** | 初始仅 1 枚硬币，每次掉落返还 2 枚 | 存活 100 秒 |
| **慢速推板** | 推板速度 ×0.4 | 达到 500 分 |
| **无侧边** | 关闭左右侧边槽位 | 达到 300 分 |
| **重力反转** | 硬币有 20% 概率向后滑动 | 达到 200 分 |
| **炸弹风暴** | 炸弹币出现率 ×5 | 存活 60 秒且得分 > 100 |
| **连锁大师** | 只有连锁掉落才计分 | 达到 10 次连锁 |

- **建议关卡数**：首发 6 关，后续可扩展
- **解锁条件**：经典模式达到指定分数后解锁

### 6.4 无尽模式（Endless）

- **定位**：长期回流内容，持续加压
- **规则**：
  - 初始 50 枚硬币
  - 每 60 秒推板速度提升 5%
  - 每 120 秒特殊币概率降低 10%
  - 硬币用完即结束
- **目标**：存活尽可能久，获得最高分
- **结算指标**：存活时间、总分、达到的最高速度档位

### 6.5 练习模式（Practice）

- **定位**：新手教学与技巧训练
- **功能**：
  - 无限硬币
  - 可暂停和慢放
  - 显示推板运动轨迹预测
  - 显示硬币受力方向
  - 物理参数可视化调试层
- **子模式**：
  - 投币位置练习（学习最佳落点）
  - 时机练习（学习推板周期节奏）
  - 连锁练习（学习制造连锁掉落）

- **推板停顿期投币**：硬币自然下滑，适合精确控制落点

- 暂停期间可切换道具卡、查看帮助
- 恢复后从暂停帧继续，无状态丢失

## 12. 技术实现难点分析

### 12.1 物理模拟

**难点**：在 DOM 环境中实现流畅的硬币物理运动

**挑战**：
- DOM 元素运动性能有限，大量硬币同时运动可能卡顿
- 硬币间碰撞检测在离散网格中需要高效算法
- 推力传递的链式计算需要控制深度

**解决方案**：
- 使用 `requestAnimationFrame` + 固定时间步长（30Hz 逻辑步）
- 碰撞检测使用空间分区（网格本身就是天然的空间分区）
- 推力传递限制最多 3 层，避免 O(n²) 计算
- 硬币运动使用 CSS `transform` 而非改变 `top/left`，利用 GPU 加速
- 同屏硬币数上限 120 枚，超出时合并为"堆叠"视觉表现

### 12.2 碰撞检测

**难点**：硬币-硬币、硬币-边界、硬币-槽位的精确碰撞

**挑战**：
- 网格离散化可能导致碰撞判定不精确
- 多硬币同时碰撞时的处理顺序
- 掉落判定的随机性需要合理控制

**解决方案**：
- 使用 8 邻域检测，覆盖所有可能的碰撞方向
- 碰撞处理按"推板推力方向"排序，先处理受力硬币
- 掉落概率基于物理状态（推力、位置、速度）动态计算，非纯随机
- 提供调试开关可视化碰撞检测区域

### 12.3 动画流畅度

**难点**：大量硬币同时运动时保持 60fps

**挑战**：
- DOM 重排重绘开销
- 多个 CSS 动画同时运行
- 特效叠加时的性能下降

**解决方案**：
- 棋盘使用 CSS Grid 布局，硬币使用绝对定位 + `transform`
- 推板运动使用 `will-change: transform` 提示浏览器优化
- 掉落特效使用 CSS 动画而非 JS 动画
- 使用 `requestAnimationFrame` 批量更新 DOM
- 低性能设备自动降级（减少特效、降低更新频率）

### 12.4 状态同步

**难点**：物理模拟状态与渲染状态的同步

**挑战**：
- 逻辑步（30Hz）与渲染帧（60fps）不同步
- 暂停/恢复时状态一致性
- 快速操作时的输入缓冲

**解决方案**：
- Simulation 与 Rendering 严格分离
- 逻辑状态使用固定时间步长更新
- 渲染层插值两帧之间的状态，实现平滑动画
- 输入队列缓冲，每逻辑步消费一次

### 12.5 Excel 伪装适配

**难点**：在保持游戏性的同时维持 Excel 外观

**挑战**：
- 3D 透视效果可能破坏 Excel 风格
- 动画效果需要克制，不能太"游戏化"
- 硬币视觉需要在"数据单元格"和"游戏硬币"之间平衡

**解决方案**：
- 默认使用 2D 平面视图（完全 Excel 风格），可选开启 3D 透视
- 硬币使用单元格背景色 + 圆形边框，而非纯游戏图形
- 推板使用 Excel 条件格式风格的高亮行
- 特效使用 Excel 风格的淡入淡出，而非粒子爆炸

- 玩家能一眼理解这是 Excel 壳下的推币机
- 投币 → 推板运动 → 掉落 → 计分 的完整流程顺畅

- 掉落反馈清晰，连锁掉落有爽感
- 从 Hub 进入到返回 Hub 的整个流程完整闭环

### 17.2 交互底线

- 局内同时提供键盘快捷键与可见按钮入口
- 投币位置指示器清晰可见
- 暂停与重开不造成状态损坏
- 输入状态在切 sheet 后正确清理

### 17.3 可维护性底线

- 物理参数集中配置，不硬编码在逻辑中
- Simulation 与 Rendering 严格分离
- 存档 schema 带版本号
- 渲染组件不直接修改游戏状态
- 所有关卡参数走内容表驱动

## 18. 当前结论

推币机作为街机经典游戏，其"投币-推板-掉落-奖励"的核心循环与 Excel 伪装风格高度适配：

- **棋盘 = 数据处理区**，**硬币 = 数据单元格**，**推板 = 滚动处理条**
- 物理系统可简化为离散网格 + 简化力学模型，在浏览器中流畅运行
- 奖励系统（特殊币、道具卡、大奖）提供持续追求目标
- 三种首发模式（经典、限时、挑战）覆盖不同玩家需求
- 按 Phase A → B → C 分阶段实施，先做可玩闭环，再逐步完善

建议实施顺序：
1. 先完成经典模式的最小可玩版（Phase A）
2. 再补充特殊币、道具卡、限时模式（Phase B）
3. 最后加入挑战模式、大奖系统、图鉴页（Phase C）
