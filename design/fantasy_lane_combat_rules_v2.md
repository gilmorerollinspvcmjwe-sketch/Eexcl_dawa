# 奇幻战线对推 v2 战斗规则规格

- 日期：2026-04-15
- 作用：定义运行时必须固定的战斗规则，供 `fantasyLaneBoardState`、`fantasyLaneCombatRules` 和测试直接实现。

---

## 1. 战场坐标

### 1.1 基础坐标

- 战场主轴使用归一化横坐标 `x = 0 ~ 100`
- 我方主堡占 `0 ~ 8`
- 我方出兵门占 `8 ~ 12`
- 中央交战区占 `12 ~ 88`
- 敌方出兵门占 `88 ~ 92`
- 敌方主堡占 `92 ~ 100`

### 1.2 四层结算

| 层编号 | 名称 | 允许单位 | 用途 |
| --- | --- | --- | --- |
| 0 | 前列 | 肉盾、近战 | 第一接敌层 |
| 1 | 中列 | 法术、短射程、突击 | 支援与切入 |
| 2 | 后列 | 弓手、投手、召唤 | 稳定远程层 |
| 3 | 空层 | 飞行单位 | 独立空战层 |

### 1.3 单位体型

| 体型 | 占用宽度 | 典型单位 |
| --- | --- | --- |
| S | 1.2 | 哥布林、弓手 |
| M | 1.6 | 狼骑、术士 |
| L | 2.2 | 狮鹫、石像卫士 |
| XL | 3.0 | 树人、食人魔、火龙 |

体型影响：

- 同层最小站位间隔
- 近战阻挡宽度
- 受击体感和碰撞范围

---

## 2. 单位分类

### 2.1 地面 / 空中

- 地面单位只会在 0 / 1 / 2 层出现
- 空中单位只会在 3 层出现
- 地面单位默认不能攻击空中
- 空中单位默认优先攻击地面后排和主堡

### 2.2 防御类型

| 类型 | 说明 | 典型目标 |
| --- | --- | --- |
| `light` | 轻型、数量多 | 哥布林、召唤物 |
| `heavy` | 重甲、正面抗打 | 重甲兵、石像卫士 |
| `swarm` | 群体、小体型 | 蝙蝠群、骷髅海 |
| `air` | 飞行目标 | 狮鹫、幼龙 |
| `structure` | 建筑 | 主堡、召唤塔 |

### 2.3 伤害类型

| 类型 | 主用途 |
| --- | --- |
| `physical` | 默认地面伤害 |
| `pierce` | 对重甲更有效 |
| `blast` | 处理成群目标 |
| `magic` | 稳定穿透防线 |
| `siege` | 主堡和大型目标 |
| `antiAir` | 反制空层 |

### 2.4 伤害系数表

| 伤害类型 \\ 防御类型 | light | heavy | swarm | air | structure |
| --- | --- | --- | --- | --- | --- |
| physical | 1.0 | 0.85 | 0.9 | 0.0 | 0.6 |
| pierce | 0.9 | 1.25 | 0.8 | 0.7 | 0.7 |
| blast | 0.8 | 0.9 | 1.4 | 0.0 | 0.8 |
| magic | 1.0 | 1.1 | 1.0 | 0.9 | 0.7 |
| siege | 0.7 | 1.0 | 0.8 | 0.0 | 1.8 |
| antiAir | 0.6 | 0.6 | 0.8 | 1.5 | 0.4 |

首发不做暴击、闪避和随机浮动伤害。

## 3. 生成与推进

### 3.1 出兵生成

- 玩家单位默认从 `x = 10` 生成
- 敌军单位默认从 `x = 90` 生成
- 新单位生成时，自动插入其所属层的最后方空位
- 若该层生成点 0.8 秒内无足够空间，则该指令留在队列等待

### 3.2 行军规则

- 地面单位向敌方主堡方向持续移动
- 空中单位在空层独立移动，不受地面阻挡
- 单位前方若存在己方单位且距离小于最小间隔，则减速至跟随
- 除带 `charge` 标签的单位外，不允许越过前方己方单位

### 3.3 阻挡规则

- 前列单位可以阻挡敌方前列和中列近战单位
- 中列近战单位若接敌，会在中列停下交战
- 后列远程不提供阻挡
- 空层单位只阻挡空层单位

---

## 4. 索敌规则

### 4.1 统一优先级

单位每次准备攻击时，按下面顺序找目标：

1. 当前已锁定目标仍有效时，继续打当前目标
2. 能直接威胁自己所在层前方的最近敌人
3. 能直接威胁己方主堡的最近敌人
4. 满足自身标签偏好的敌人
5. 若无可攻击单位，则继续推进

### 4.2 近战单位

- 只会寻找自己攻击范围内最近的有效目标
- 默认只能打地面
- 带 `antiAir` 的近战单位可以打低空空层目标
- 带 `charge` 的近战单位首次接敌前可忽略一次小体型友军阻挡

### 4.3 远程单位

- 优先攻击自己射程内、距离最近的有效目标
- 若单位带 `focusAir`，则空层目标优先级提升一级
- 若单位带 `siegeFocus` 且前方无敌方单位，可直接攻击主堡

### 4.4 法术与范围兵

- 默认选最近目标
- 若带 `cluster` 标签，则在射程内选择“周围敌人数最多”的目标点
- 溅射半径内的目标按同一伤害类型分别结算

### 4.5 主堡索敌

- 主堡不主动攻击
- 关卡脚本允许 Boss 关给主堡挂临时防御炮台，但不属于 v2 首发默认规则

---

## 5. 攻击与伤害结算

### 5.1 攻击循环

每个单位有：

- `attackIntervalSec`
- `windupSec`
- `recoverSec`

默认规则：

- `windupSec = attackIntervalSec * 0.25`
- `recoverSec = attackIntervalSec * 0.35`
- 剩余时间视为稳定输出段

首发运行时可以只用 `attackIntervalSec`，但数据上要预留这 3 个字段。

### 5.2 护甲与生命

- 伤害先结算到 `armorHp`
- `armorHp` 归零后，剩余伤害继续打 `hp`
- 无护甲的单位直接掉 `hp`
- 首发不做“护甲值减伤公式”，只做护甲血双段结构

### 5.3 伤害公式

```text
finalDamage =
  baseDamage
  * damageTypeMultiplier
  * sourceBuffMultiplier
  * targetDebuffMultiplier
```

首发不做随机系数。

### 5.4 范围伤害

- `splashRadius = 0` 时只打主目标
- `splashRadius > 0` 时，命中点周围单位按同类型伤害计算
- 溅射不区分敌我时，必须在脚本或技能定义中显式标记 `friendlyFire`
- 首发默认无友伤

## 6. 状态效果

首发只允许 6 类状态，禁止继续扩散：

| 状态 | 效果 |
| --- | --- |
| `burn` | 持续掉血 |
| `slow` | 移速和攻速下降 |
| `freeze` | 短时间完全停住 |
| `shield` | 获得额外护盾血 |
| `haste` | 移速和攻速上升 |
| `mark` | 受到额外伤害 |

规则：

- `freeze` 优先级高于 `slow`
- 相同状态默认刷新持续时间，不叠层
- `burn` 可叠 2 层
- `mark` 不叠层，只刷新

## 7. 技能规则

### 7.1 技能分类

- 英雄主动技能
- 通用战术技能
- Boss 脚本技能

### 7.2 施放约束

- 技能不会暂停战斗
- 技能有明确前摇提示
- 技能目标区域只允许以下 4 种：
  - 直线
  - 圆形区域
  - 友方前线区域
  - 敌方主堡区域

### 7.3 技能设计边界

- 技能不能直接永久召唤高费兵
- 技能不能重置全部单位冷却
- 技能不能让玩家跳过读题，必须服务于保线、反打或终结

---

## 8. 主堡规则

### 8.1 主堡属性

- 我方主堡基础生命：`3000`
- 敌方主堡基础生命：`3000`
- 章节难度允许按关卡加成 `0% ~ 40%`

### 8.2 拆堡规则

- 单位进入敌方主堡攻击区后，如果前方无有效敌方单位，才开始攻击主堡
- 优先打单位，后打主堡
- 带 `siegeFocus` 的单位在满足条件时会优先打主堡

### 8.3 胜负判定

- 敌方主堡生命归零：胜利
- 我方主堡生命归零：失败
- 时间到仍未结束：进入 45 秒终局加压阶段
- 终局加压阶段结束仍未结束：按我方失败处理

---

## 9. 终局加压阶段

触发时间：战斗时间达到 `battleTimeLimitSec`

效果：

- 双方资源回复 `+30%`
- 全体单位移速 `+15%`
- 英雄与战术技能剩余冷却减半
- 画面进入红色告警态

目的：

- 防止无聊拖线
- 强制进入决战
- 让落后方仍有一次翻盘窗口

---

## 10. 必测规则清单

- 同层阻挡是否正确
- 空层与地面层是否独立结算
- 护甲血和本体血是否按顺序结算
- 对空、群伤、破甲的伤害系数是否生效
- 主堡攻击优先级是否晚于单位攻击
- 终局加压阶段是否能正常触发与收口
