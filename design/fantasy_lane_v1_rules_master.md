# 奇幻战线 v1 规则总表

- 日期：2026-04-16
- 状态：可执行基线
- 用途：作为 `fantasy_lane` 运行时、兵种表、关卡表、UI 表现和测试的统一规则源。后续新增兵种或关卡，默认不得突破本规则总表，除非先更新本文档。

---

## 1. v1 设计目标

v1 不追求复杂机制堆叠，优先保证以下五点：

1. 战场一眼能看懂，玩家能区分空中、地面、远程、近战和主堡压力。
2. 近战、远程、空军三套行为逻辑必须明确分离，不共享一套模糊规则。
3. 兵种、关卡、技能都能通过表驱动扩展，而不是频繁返工底层。
4. UI 是 Excel 风格外壳，信息密度高但不遮挡主战场。
5. 运行时指标可统计、可调试、可回放推演。

## 2. 战场总规则

### 2.1 战场结构

| 项目 | v1 规则 |
| --- | --- |
| 战场结构 | 单主战场，无硬分路 |
| 层级 | `ground` 地面层、`air` 空中层 |
| 横向主轴 | `x = 0 ~ 100` |
| 我方主堡位置 | `x = 3` |
| 敌方主堡位置 | `x = 97` |
| 我方出兵点 | `x = 8` |
| 敌方出兵点 | `x = 92` |
| 交战中心 | 根据双方地面最前线实时计算 |
| 玩家人口上限 | 默认 `18` |
| 指令队列长度 | 默认 `5` |
| 玩家全局出兵 CD | `320ms` |
| 敌军全局出兵 CD | `420ms` |
| 金币上限 | `320` |
| 终局加压 | 超过时限后进入 `45s` 终局倒计时 |

### 2.2 层级说明

| 层级 | 描述 | 主要单位 |
| --- | --- | --- |
| `ground` | 地面主战区，负责正面交战、堵线、攻城 | 坦克、近战、法师、弓手、攻城器 |
| `air` | 空中战区，独立移动，不被地面阻挡 | 狮鹫、幼龙、蝙蝠群、火龙 |

### 2.3 主战场可读性

1. 地面和空中必须在视觉上明显分区。
2. 主堡区域、出生区、交战中心必须有稳定标记。
3. 主战场中心区域禁止放大卡片式 UI。

## 4. 单位行为协议

### 4.1 行为总原则

1. 单位先按兵种行为模型决定移动与作战姿态。
2. 友军碰撞是软限制，不允许整排冻结。
3. 当前目标死亡时，必须立即进入重选目标或继续推进状态。
4. 单位不能因为缺少完美站位而长期原地发呆。

### 4.2 近战单位规则

1. 只要发现可攻击目标，持续向目标推进。
2. 进入攻击距离后停下攻击。
3. 目标死亡后立即重新索敌。
4. 当前没有目标时，持续向敌方主堡方向推进。
5. 近战不会主动后撤。

### 4.3 远程单位规则

1. 远程单位拥有独立的 `preferredRange` 和 `minimumRange`。
2. 发现目标但未进入有效攻击区时，向前移动到理想射程。
3. 进入有效攻击区后停下输出。
4. 敌人贴入 `minimumRange` 内时，可轻微后撤或纵向让位。
5. 目标死亡后，若新目标已在射程内则直接切目标开火，否则继续移动到射程内。
6. v1 默认不做边走边打。

### 4.4 空军单位规则

1. 空军不受地面单位碰撞阻挡。
2. 空军只受空军软碰撞约束。
3. 可空地双打的单位在空中目标存在时，对高威胁空军有优先权重。

### 4.5 换目标规则

| 项目 | v1 规则 |
| --- | --- |
| 当前目标保留 | 目标仍存活且仍可打时优先保留 |
| 换目标延迟 | `80ms - 160ms` |
| 索敌频率 | 每个逻辑 tick 可重评估 |
| 目标评分 | 距离 + 威胁 + 人口 + 克制权重综合 |

### 4.6 高威胁目标

以下目标拥有额外被集火权重：

1. `caster`
2. `siege`
3. `finisher`
4. 高人口空军

## 5. 运动、碰撞与站位

### 5.1 友军碰撞

| 项目 | v1 规则 |
| --- | --- |
| 地面友军碰撞 | 软碰撞 |
| 空军友军碰撞 | 软碰撞 |
| 同轨迹判断 | 只有纵向重叠足够接近才互相卡位 |
| 后排绕行 | 允许轻微纵向漂移 |
| 巨型单位阻挡 | 更强 |
| 小体型穿插 | 更容易 |

### 5.2 敌我接敌

1. 敌我接敌后，近战以接敌面为主形成阻挡。
2. 远程单位不承担完整阻挡职责。
3. 巨型单位可显著压缩道路。
4. 空军与地面阻挡完全分离。

### 5.3 禁止事项

以下行为在 v1 明确禁止：

1. 因为地面层存在友军而导致整排后军永久不动。
2. 所有单位共享完全一致的碰撞半径。
3. 空军被地面单位静态卡住。

## 6. 攻击流程

### 6.1 标准攻击阶段

1. 索敌
2. 判定是否进入可攻击区
3. 进入前摇
4. 前摇结束时出手
5. 近战即时命中或远程发射投射物
6. 进入攻击间隔

### 6.2 攻击锁脚

| 项目 | v1 规则 |
| --- | --- |
| 近战攻击锁脚 | 有 |
| 远程攻击锁脚 | 有 |
| 锁脚时长 | 主要由 `attackAnimMs` 决定 |
| 攻击打断 | v1 不做复杂打断体系 |

### 6.3 投射物

1. 所有远程单位默认使用投射物结算。
2. 投射物飞行时间由 `projectileSpeed` 决定。
3. 投射物允许命中单位或主堡。
4. AOE 投射物命中后再做范围伤害。

## 7. 伤害与护甲体系

### 7.1 护甲类型

| 类型 | 描述 |
| --- | --- |
| `light` | 轻甲、人形脆皮 |
| `heavy` | 重甲、重型单位 |
| `swarm` | 集群、小体型单位 |
| `structure` | 建筑级或攻城级目标 |
| `air` | 飞行目标，作为目标判定键使用 |

### 7.2 伤害类型

| 类型 | 主用途 |
| --- | --- |
| `physical` | 默认地面伤害 |
| `pierce` | 对重甲更有效 |
| `blast` | 处理抱团和群体 |
| `magic` | 稳定法术伤害 |
| `siege` | 对主堡和大体型更强 |
| `antiAir` | 对空军更强 |

### 7.3 克制表

| 伤害类型 \\ 防御类型 | light | heavy | swarm | air | structure |
| --- | --- | --- | --- | --- | --- |
| physical | 1.00 | 0.82 | 0.90 | 0.00 | 0.58 |
| pierce | 0.88 | 1.25 | 0.76 | 0.72 | 0.70 |
| blast | 0.80 | 0.92 | 1.42 | 0.90 | 0.82 |
| magic | 1.00 | 1.08 | 1.00 | 0.95 | 0.72 |
| siege | 0.68 | 0.95 | 0.84 | 0.45 | 1.85 |
| antiAir | 0.70 | 0.76 | 0.90 | 1.55 | 0.40 |

### 7.4 AOE 规则

1. AOE 默认圆形。
2. 中心命中吃满伤害。
3. 外圈按距离衰减。
4. 最低伤害不低于中心值的 `35%`。
5. 次级 AOE 默认基于主伤害的 `48% * 衰减系数`。

## 8. 经济与出兵规则

### 8.1 经济规则

| 项目 | v1 规则 |
| --- | --- |
| 开局金币 | 按关卡定义 |
| 金币上限 | `320` |
| 基础收入 | 前期 `9.5/s`，中期 `11/s`，后期 `12.5/s`，超后期 `14/s` |
| 囤钱衰减 | `230+` 降收益，`290+` 极低收益 |

### 8.2 出兵规则

| 项目 | v1 规则 |
| --- | --- |
| 扣费时机 | 下达指令即扣费 |
| 队列长度 | 默认 `5` |
| 单位冷却 | 各兵种独立 |

| 人口超限 | 不允许部署 |

### 8.3 费用分层建议

| 层级 | 费用区间 | 人口 | CD |
| --- | --- | --- | --- |
| 低费铺线 | `35 - 55` | `1` | `3 - 4s` |
| 中费功能 | `70 - 120` | `2 - 3` | `5 - 9s` |
| 高费转折 | `120 - 180` | `3 - 4` | `9 - 14s` |

## 9. 兵种定位模板

| 角色 | 职责 | 典型特征 |
| --- | --- | --- |
| `tank` | 稳线、吃伤害、堵路 | 高 HP、高护甲、低速 |
| `fighter` | 突击、切入、压前 | 中速、中近战、高接敌积极性 |
| `sniper` | 点杀高价值目标 | 远射程、单体、高 DPS、脆 |
| `caster` | 群伤或控制 | 中远程、AOE、脆、法术特效明显 |
| `siege` | 打主堡和大体型 | 慢、重投射、高结构伤害 |
| `air_sup` | 抢空优 | 空军、对空加权 |
| `finisher` | 后期收口 | 高费、高人口、强压制 |

### 9.1 编组风险检查标准

| 检查项 | 警告条件 |
| --- | --- |
| 前排人口 | 小于 `4` |
| 对空人口 | 推荐对空关卡中小于 `3` |
| AOE 人口 | 推荐群伤关卡中小于 `2` |
| 高费负重 | 高费人口过高时给出“编组过重”警告 |

## 10. 技能规则

### 10.1 技能分类

| 类型 | 用途 |
| --- | --- |
| 英雄技能 | 改变战场节奏，偏强，CD 长 |
| 战术技能 | 解决当前节点问题，偏功能，CD 略短 |

### 10.2 v1 技能边界

1. 不做需要手动拖选大范围的复杂微操。
2. 允许全局、交战中心、指定层三种作用方式。
3. 技能不能替代兵种体系本身。
4. 技能必须有明确预警、颜色和命中反馈。

---

## 11. 胜负、时长与阶段

### 11.1 胜负条件

| 项目 | v1 规则 |
| --- | --- |
| 胜利 | 敌方主堡 HP 归零 |
| 失败 | 我方主堡 HP 归零 |
| 超时处理 | 进入终局加压，不直接结束 |
| 终局失败 | 终局倒计时结束仍未推平敌堡 |

### 11.2 结算与星级

1. 通关必定有星级。
2. 高主堡血量可提高星级。
3. 拖入终局加压通常降低高星空间。

### 11.3 关卡结构

普通关卡默认三段：

1. 开场读题段
2. 中段对推段
3. 尾段加压段

Boss 关额外包含：

1. HP 阈值阶段
2. 阶段预警
3. 阶段召唤
4. 阶段强化效果

## 12. UI 与信息展示规则

### 12.1 战斗界面必须常驻显示

1. 金币
2. 人口 `activePop / popLimit`
3. 我方主堡 HP
4. 敌方主堡 HP
5. 当前阶段
6. 前线位置
7. 空优差值
8. 拥堵度
9. 指令队列
10. 英雄技能与战术技能冷却

### 12.2 禁止遮挡战场的常驻信息

1. 长段说明文字
2. 大卡片式任务面板
3. 永久展开的兵种词典
4. 占据中心的大面积结果或提示面板

### 12.3 视觉方向

1. UI 外壳使用 Excel / 工作簿风格。
2. 主战区禁止暖黄色大卡片风格。
3. 战场单位主体以程序化 sprite 为核心，不使用 emoji。

## 14. 章节与关卡设计协议

每章只教 1 到 2 个核心命题：

| 章节 | 主命题 |
| --- | --- |
| 1 | 前排、群伤、基础铺线 |
| 2 | 持续召唤与拖线 |
| 3 | 对空与控场 |
| 4 | 破甲、攻城、高费转折 |
| 5 | 空地混编与终局收口 |

### 14.1 内容原则

1. 低费兵负责铺线。
2. 中费兵负责转折与功能。
3. 高费兵负责后期收口。
4. 不在前两章同时压入全部复杂机制。

---

## 15. 平衡与制作约束

### 15.1 强制平衡原则

1. 没有任何单卡能解决全部问题。
2. 对空、AOE、破甲、终结在整套内容中都必须有真实需求。
3. 高费兵强，但不能过早压死前中期。
4. 巨型单位强阻挡，但不能无限堆叠。

### 15.2 新兵种准入规则

新增兵种进入 v1 主线前，必须回答：

1. 它属于哪种角色模板。
2. 它解决什么战术问题。
3. 它被什么克制。
4. 它的费用、人口、速度、射程和体型落在哪个层级。
5. 它是否会破坏现有碰撞与索敌规则。

---

## 16. 调试与统计要求

开发期必须保留以下运行时统计：

1. 每局召唤数
2. 每局击败数
3. 投射物发射数
4. AOE 命中数
5. 前排人口投放量
6. 对空人口投放量
7. 金币高位停留时间
8. 拥堵时间
9. 技能释放次数与时间点
10. 单位平均接敌时间

---

## 17. 实施优先级

按以下顺序执行，不允许倒置：

1. 行为协议
2. 碰撞与推进逻辑
3. 投射物与命中表现
4. 战斗 HUD 与边缘信息布局
5. 兵种数值批量调优
6. 新技能、新兵种、新关卡扩展

---

## 18. v1 冻结结论

本版明确冻结以下关键决策：

1. 单主战场，不做硬分路。
2. 空地分层，行为独立。
3. 近战必须持续推进到接敌。
4. 远程必须按独立射程圈作战。
5. 友军碰撞为软碰撞，不允许整排冻结。
6. 护甲克制和伤害类型固定。
7. 人口上限 `18`，队列 `5`，金币上限 `320`。
8. UI 使用 Excel 外壳，不用暖色卡片式战场。
9. 投射物必须按兵种类型区分形状。
10. 后续新增兵种必须遵守本标准字段和模板。
