# 黄金矿工（Gold Miner）Excel 游戏详细设计

- 日期：2026-04-14
- 项目：`excel-aim-trainer`
- 设计目标：在现有 Excel 风格游戏中心中高还原交付经典 Gold Miner（钩子摆动抓取 + 限时达标）体验，并预留章节化扩展能力
- 设计阶段：实施前详细设计（可直接拆解任务）

---

## 1. 游戏概述

### 1.1 一句话介绍

黄金矿工是一个"看准时机甩钩子，抓到值钱东西就赚"的休闲游戏——在 Excel 表格里，它变成了"数据抓取器"，钩子是公式引用线，抓到的是各种数据单元格。

### 1.2 核心卖点

- **3 秒可理解**：钩子来回摆，按空格甩出去，抓到东西拉回来算分
- **时机判断 + 风险决策**：大金子值钱但拉得慢，小石头不值钱还拖时间
- **短局高压循环**：每关 60 秒左右，目标分数逐关递增，失败即重开
- **商店策略层**：关间用赚到的钱买道具，形成"赚 -> 投资 -> 赚更多"的正循环
- **Excel 伪装天然适配**：钩子 = 单元格引用线，物品 = 数据单元格，得分 = 求和公式

## 2. 原版游戏分析

### 2.1 原版核心机制

| 机制 | 说明 |
| --- | --- |
| 钩子摆动 | 钩子从屏幕顶部中心点以固定角度范围（约 ±80°）来回匀速摆动 |
| 发射钩子 | 玩家按键后，钩子沿当前摆动方向直线射出 |
| 抓取物品 | 钩子碰到物品后停止延伸，抓住物品并沿原路收回 |
| 重量影响速度 | 物品越重，收回速度越慢；轻物品快速收回，重物品缓慢拖回 |
| 限时达标 | 每关有倒计时，时间结束前分数达到目标值则过关 |
| 关间商店 | 过关后可用当前金钱购买道具，辅助后续关卡 |

### 2.2 原版玩法循环

```
关卡开始
  -> 钩子自动摆动
  -> 玩家判断时机，按键发射
  -> 钩子直线延伸
  -> 碰到物品 -> 抓住 -> 收回（速度受重量影响）
  -> 物品入库，加分
  -> 钩子回到起点，继续摆动
  -> 倒计时结束
  -> 分数 >= 目标 -> 过关 -> 进入商店 -> 下一关
  -> 分数 < 目标 -> 失败 -> 重开本关
```

## 3. 核心玩法规则

### 3.1 钩子摆动

- 摆动中心：游戏区域顶部中央
- 摆动角度范围：-80° 到 +80°（以垂直向下为 0°）
- 摆动速度：匀速，约 60°/秒（完整摆动周期约 2.7 秒）
- 摆动方向：从左到右再从右到左，循环往复
- 视觉表现：一条从起点延伸的线段，末端带钩子图标

### 3.2 发射与抓取

- 发射触发：玩家按下空格键或鼠标左键
- 发射方向：钩子当前摆动角度的直线方向
- 延伸速度：固定速度，约 400 像素/秒
- 最大延伸距离：到达游戏区域底部边界
- 抓取判定：钩子末端与物品碰撞盒相交即触发抓取
- 抓取优先级：当钩子同时碰到多个物品时，取距离钩子末端最近的物品

### 3.3 收回机制

- 收回路径：沿发射方向原路返回
- 收回速度：基础速度 / 物品重量系数
- 空钩收回：未抓到物品时，以最大速度收回
- 收回完成：钩子回到起点后，物品价值计入总分，钩子恢复摆动

### 3.4 得分计算

```
单局得分 = Σ(被抓取物品价值)
过关条件：单局得分 >= 关卡目标分数
过关奖励：当前总金钱 += 单局得分
失败惩罚：总金钱不变，重开本关
```

### 3.5 炸药机制（可选操作）

- 使用条件：钩子正在收回且持有炸药道具
- 触发方式：按下上方向键或鼠标右键
- 效果：炸掉当前钩子上的物品，钩子立即加速收回
- 消耗：每次使用消耗 1 个炸药

## 7. 操作流程

### 7.1 键盘映射

| 键位 | 动作 | 适用场景 |
| --- | --- | --- |
| `Space` / `鼠标左键` | 发射钩子 | 钩子摆动中 |
| `Up` / `鼠标右键` | 使用炸药 | 钩子收回中且有炸药 |
| `P` | 暂停 / 继续 | 游戏中 |
| `R` | 重开本关 | 游戏中 / 失败后 |
| `Esc` | 返回 Hub | 暂停状态 |
| `1~8` | 快捷购买商店道具 | 商店界面 |
| `Enter` | 确认购买 / 进入下一关 | 商店界面 / 过关后 |

### 7.2 状态机

```
idle --(Space/点击开始)--> swinging
swinging --(Space/左键)--> extending
extending --(触底/碰到物品)--> retracting
retracting --(回到起点)--> swinging
retracting --(Up/右键 + 有炸药)--> dynamite -> retracting(加速)
swinging --(时间到)--> check_result
check_result --(分数>=目标)--> shop
check_result --(分数<目标)--> game_over
shop --(Enter/点击下一关)--> idle(下一关)
game_over --(R/点击重开)--> idle(本关)
swinging --(P)--> paused
paused --(P)--> swinging
paused --(Esc)--> idle
```

### 7.3 游戏状态定义

```ts
type GoldMinerStatus =
  | 'idle'        // 待开始
  | 'swinging'    // 钩子摆动中
  | 'extending'   // 钩子延伸中
  | 'retracting'  // 钩子收回中
  | 'paused'      // 暂停
  | 'shop'        // 商店界面
  | 'level_complete'  // 过关
  | 'game_over'   // 失败
```

## 11. 技术实现难点分析

### 11.1 钩子物理系统

**难点**：钩子的摆动、发射、收回三态切换需要精确的角度和位置计算。

**方案**：
- 摆动状态：使用角度变量 `hookAngle`，每帧增加 `angularVelocity`，到达边界后反向
- 发射状态：根据当前角度计算方向向量 `(sin(angle), cos(angle))`，钩子末端位置 = 起点 + 方向 × 延伸距离
- 收回状态：沿发射方向的反方向移动，延伸距离递减
- 关键：三态切换时保持位置连续性，避免跳变

### 11.2 碰撞检测

**难点**：钩子末端（一个点）与多种形状物品的碰撞判定。

**方案**：
- 所有物品使用圆形碰撞盒（简化计算）
- 碰撞判定：钩子末端到物品中心的距离 < 物品碰撞半径
- 多物品碰撞：取距离最近的物品
- 性能优化：只检测钩子末端前方扇形区域内的物品

### 11.3 物品布局生成

**难点**：每关物品位置需要合理分布，既不能太简单也不能不可能完成。

**方案**：
- 使用预定义模板 + 随机扰动
- 每个关卡包有若干布局模板
- 模板定义物品类型、大致区域、数量范围
- 生成时在模板基础上随机偏移位置
- 确保：钻石和大金块不会放在角落死角

### 11.4 动态物品 AI

**难点**：鼹鼠和蝙蝠的移动路径需要自然且可预测。

**方案**：
- 鼹鼠：水平匀速移动，碰壁反向，速度固定
- 蝙蝠：正弦波飞行 `y = baseY + amplitude * sin(frequency * time)`，水平方向匀速
- 被抓取后立即停止移动，转为普通物品

### 11.5 渲染性能

**难点**：游戏区域可能有 20-40 个物品，加上钩子和特效，需要保持 60fps。

**方案**：
- 使用 Canvas 渲染游戏主区域（非 DOM 网格）
- 物品使用简单几何图形（圆形、方形）
- 钩子使用线段绘制
- 特效使用粒子系统（简化版）
- 只在状态变化时重绘，不每帧全量重绘

### 11.6 与 Excel 网格的协调

**难点**：游戏主区域使用 Canvas，但需要与 Excel 网格风格协调。

**方案**：
- Canvas 背景绘制 Excel 网格线
- 物品渲染为"单元格"样式（带边框的方块）
- 钩子渲染为"引用线"样式（虚线）
- HUD 使用 DOM 元素，保持 Excel 表格外观

## 16. 技术实现边界

### 16.1 渲染方案选择

黄金矿工游戏区域建议使用 **Canvas** 而非 DOM 网格，原因：
- 钩子是连续角度的线段，不适合离散网格表达
- 物品位置是连续坐标，不是网格对齐
- 需要高频重绘（60fps），Canvas 性能更好
- 可以在 Canvas 背景绘制 Excel 网格线保持风格统一

HUD、覆盖层、商店等 UI 继续使用 DOM/React 组件。

### 16.2 Simulation 与 Rendering 边界

- **Simulation**：钩子角度/位置、物品位置、碰撞判定、计分、关卡推进
- **Rendering**：Canvas 绘制钩子、物品、特效；DOM 渲染 HUD、覆盖层
- **Tick**：固定步进（建议 60Hz 逻辑步），渲染跟随浏览器帧率

### 16.3 数据与存档边界

- 仅存可序列化状态，不存 Canvas 运行态
- 存档分为：`settings`、`progress`、`activeRun`
- 所有关卡参数走内容表，避免逻辑分支硬编码

## 18. 新手引导设计

### 18.1 首次引导（3 步以内）

1. **看钩子摆动**：钩子会自动左右摆动，注意它的方向

引导要求：
- 不打断操作节奏
- 能在 5 秒内看懂
- 第一次完成后不再强弹

### 18.2 失败后回流

- `再来一把`
- `返回 Hub`
- `查看记录`

如果本局刷新了记录，明确显示：
- 新最高分
- 新最高关卡
- 首次抓到钻石

## 20. 当前结论

黄金矿工适合作为下一个实施的 arcade 模块，原因是：

- 核心机制清晰（摆动 -> 发射 -> 抓取 -> 收回），状态机简单
- 与现有模块的"状态层 + UI 壳层"模式完全兼容
- 单局时长可控（30-60 秒），与现有模块切换节奏一致
- 关卡数据驱动，易于扩展和调优
- Canvas 渲染游戏区域 + DOM 渲染 HUD 的混合方案成熟可行
- 高可扩展：同一运行时可扩展无尽模式、挑战模式、训练模式

建议按本设计推进，优先完成 `Phase A`，先让 `Sheet18` 跑通基础循环，再补商店、音效和完整关卡。
