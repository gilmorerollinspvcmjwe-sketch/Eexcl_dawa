# 消消乐（Match-3）Excel 游戏深度设计

- 日期：2026-04-14
- 项目：`Eexcl_dawa`
- 设计目标：在当前 Excel 风格游戏中心中高还原经典 Match-3 / Bejeweled / 开心消消乐系核心体验，并沉淀可持续扩展的关卡与模式体系
- 设计阶段：实施前详细设计（可直接拆任务）

---

## 1. 设计目标与原版体验支柱

### 1.1 设计目标

- 提供一个与 PvZ（防线经营）、Snake（路径生存）、Tetris（堆叠整理）不同的“短反馈+连锁爽感”模块。
- 让玩家在 5 秒内理解操作规则（交换相邻格），在 30 秒内进入“找形状、做连锁、冲目标”的稳定循环。
- 首发即支持可扩展关卡脚本，不做只可演示的固定棋盘 demo。
- 在 Excel 外壳中保持原版感：规则硬核还原，但信息呈现和 UI 语义使用工作簿语境。

### 1.2 原版体验支柱（必须保留）

1. **交换必须有意义**：只允许触发消除的相邻交换，非法交换回弹。
2. **三消判定清晰**：横向/纵向连续 >=3 即消除，视觉反馈和计分规则稳定。
3. **掉落与连锁是核心快感**：一次操作可触发多段 cascade，后段连锁分值递增。
4. **特殊块组合决定上限**：四连、五连、T/L 形生成强力块，块与块组合有高爆发策略。
5. **目标驱动而非纯分数**：步数、收集、清障、下落等目标让关卡有明确解题方向。
6. **失败可解释**：耗尽步数/时间未达成目标时，系统能给出清晰失败原因和改进建议。

---

## 2. 为什么适合当前游戏中心与 Sheet 职责建议

> 本文中的 Sheet 编号按当前新增三套经典模块的统一接入方案固定为 `Sheet16 / Sheet17`，避免与 Pac-Man、Zuma 的模块页冲突。

### 2.1 为什么适合放进当前合辑

- 与现有模块形成节奏互补：Match-3 更强调“短决策 + 连锁反馈 + 目标解题”。
- Excel 网格天然适配三消棋盘表达，视觉语言迁移成本低。
- 记录层可复用现有经验：关卡进度、最佳分数、星级、每日挑战与周目标均可沿用现有存储结构思路。
- 非常适合作为“可持续内容位”：每次扩展可新增关卡包，不必改底层玩法。

### 2.2 Sheet 职责建议

- **Sheet16（主玩法）**：选关、开局、战斗、HUD、结算。
- **Sheet17（图鉴/训练）**：特殊块说明、障碍规则、组合实验场、练习关入口。
- **Sheet1（Hub）**：显示 Match-3 入口卡、进度摘要（已通关、星级、今日挑战状态）。

## 3. 核心玩法系统（高还原基线）

### 3.1 核心 loop

1. 玩家选择关卡（目标/步数/障碍已知）。
2. 进入棋盘后执行一次相邻交换。
3. 若形成消除，进入结算链：消除 -> 掉落 -> 补牌 -> 连锁判定。
4. 更新目标进度、步数或时间、分数与连锁倍率。
5. 达成目标则胜利；资源耗尽未达标则失败。

### 3.2 棋盘与元素规格（首发建议）

- 基础棋盘：`8x8`（教程关可用 `7x7`，高压关可用 `9x9`）。
- 基础颜色：5~6 种（首发建议 6 种，便于控制概率与难度）。
- 每局保证存在合法交换；若无合法交换则自动洗牌（保留障碍状态）。

### 3.3 交换规则

- 仅允许曼哈顿相邻交换（上下左右）。
- 非法交换：动画回弹，不扣步数。
- 合法交换：至少触发一个消除群组才提交操作并扣步数（步数关）。
- 输入优先级：`swap > resolve > inputLock`，结算中禁二次输入，防状态穿透。

### 3.4 判定、掉落与连锁

- 同步扫描横纵连续段，标记所有待消格。
- 一次结算可同时存在多个群组，统一清除后再掉落。
- 掉落顺序：从下到上逐列压缩，空位由顶部补牌。
- 连锁定义：同一次玩家操作触发的第 N 次自动消除即第 N 段连锁。
- 连锁倍率：建议 `1.0 -> 1.25 -> 1.5 -> 1.75 -> 2.0` 上限，便于分数曲线可控。

### 3.5 特殊块体系（首发高还原）

- **四连（横/纵）**：生成条纹块（行清/列清）。
- **五连直线**：生成彩球（颜色炸弹）。
- **T/L 形五连**：生成包装块（3x3 爆炸）。
- **组合规则（必须首发）**：
  - 条纹 + 条纹：十字双清。
  - 条纹 + 包装：三行三列爆破。
  - 包装 + 包装：大范围双爆。
  - 彩球 + 普通色块：清屏该颜色。
  - 彩球 + 特殊块：全色转化为对应特殊块并触发。

### 3.6 目标类型

- 分数达标（Score Target）。
- 颜色收集（Collect Colors）。
- 清理覆盖层（Clear Jelly / Frost）。
- 掉落收集（Drop Ingredients / 节点物件到底）。
- 障碍清除（Break Crates / Chains / Stones）。
- 复合目标（如“20 步内清理果冻并收集 10 红色块”）。

### 3.7 障碍体系（首发建议）

- 冻结层（需 1~2 次消除击破）。
- 锁链块（被锁格不可移动，消除邻接后解锁）。
- 木箱/石块（不可交换，不可下落，需邻接爆破）。
- 传送口（入口/出口映射，形成非线性掉落路径）。
- 巧克力/蔓延块（每回合扩散，逼迫玩家主动处理）。

### 3.8 失败条件

- 步数耗尽且目标未达成。
- 时间耗尽且目标未达成（计时模式）。
- 关键目标未完成（如收集物未到达底部）。
- 特殊事件失败（后续可扩：蔓延超过阈值）。

### 3.9 节奏与难度曲线（原版感）

- 前 10 关：单目标 + 低障碍，建立交换与特殊块直觉。
- 11~30 关：引入复合目标和一层障碍，要求规划步数。
- 31~60 关：障碍组合 + 掉落目标 + 限步压力，逼迫连锁效率。
- 61+ 关：高密障碍与特殊块联动关，强调“先解场，再冲目标”。

## 4. 模式、关卡结构与难度分层（原版还原导向）

### 4.1 首发模式矩阵

- **Adventure（主线闯关）**：以步数和目标为主，关卡式推进，核心模式。
- **Blitz（限时冲分）**：60s/120s，追求高分与高连锁。
- **Zen（练习模式）**：无失败或弱失败，用于熟悉特殊块组合。
- **Puzzle（固定盘解题）**：给定棋盘与目标步数，解法导向。

### 4.2 关卡脚本结构建议

每关最少定义：

- `boardTemplateId`
- `palette`（颜色集合）
- `moveLimit` 或 `timeLimitMs`
- `targets[]`（类型、数量、优先级）
- `blockers[]`（类型、初始坐标、层数）
- `spawnRules`（补牌概率、特殊块注入策略）
- `tutorialHint`（可选）

### 4.3 难度控制旋钮

- 颜色种类数量（5/6/7）。
- 初始合法交换数量阈值。
- 障碍密度与层数。
- 目标数与目标组合复杂度。
- 步数预算（目标理论最优步数的 1.3~2.2 倍）。
- 关键颜色生成权重（防止“目标色枯竭”）。

## 5. 正式内容包拆分（8 包）

1. **新手入门包：三消与交换基础**
   - **定位**：建立“只交换有效步、三消判定、基础连锁”的绝对基本功。
   - **原版参考**：Bejeweled 早期教学、开心消消乐前十关引导。
   - **核心规则**：仅分数目标与颜色收集目标；障碍最小化；突出非法交换回弹与合法交换反馈。
   - **建议关卡数**：12 关。
   - **首发必做**：教学提示、可视化判定高亮、基础结算星级（1~3 星）。
   - **可延期内容**：动态教学语音、自动建议步提示系统。
   - **Excel 对应**：`Sheet16` 默认章节；公式栏显示“本回合目标/剩余步数”。

2. **步数策略包：有限步目标闯关**
   - **定位**：让玩家开始做资源管理，强调“每一步都要赚”。
   - **原版参考**：Candy Crush Saga 核心步数关。
   - **核心规则**：固定步数 + 单/双目标；引导玩家优先制造特殊块提升效率。
   - **建议关卡数**：20 关。
   - **首发必做**：步数倒计时 HUD、失败原因拆解（差目标数、浪费步比例）。
   - **可延期内容**：关后“最佳 3 步复盘”提示。
   - **Excel 对应**：状态栏突出 `剩余步数`；结算页显示“步效评分”。

3. **果冻清理包：覆盖层拆除**
   - **定位**：从“只凑颜色”升级到“区域清理”。
   - **原版参考**：Candy Crush 果冻关、Toon Blast 覆盖层关。
   - **核心规则**：需要在指定区域触发消除，分层覆盖需要多次击破。
   - **建议关卡数**：18 关。
   - **首发必做**：覆盖层可视化（层数、剩余格），区域高亮。
   - **可延期内容**：双层/三层混合覆盖与区域连锁挑战。
   - **Excel 对应**：`Sheet17` 图鉴展示“覆盖层被清除条件”；`Sheet16` HUD 增加“剩余覆盖格”。

4. **下落收集包：物件护送到底**
   - **定位**：引入路径规划与列控制，强调底部出口与清路。
   - **原版参考**：Candy Crush 掉落材料关、Farm Heroes 收集路径关。
   - **核心规则**：特殊收集物由顶部进入，需通过消除让其落到底部出口。
   - **建议关卡数**：16 关。
   - **首发必做**：出口格高亮、收集物轨迹提示、失败时卡点复盘。
   - **可延期内容**：多出口关、障碍封路关、限时护送关。
   - **Excel 对应**：`Sheet16` 棋盘增加出口标记；状态栏显示“待下落物件数”。

5. **障碍拆解包：锁链、木箱与蔓延压制**
   - **定位**：建立“先解场再做目标”的中后期节奏。
   - **原版参考**：开心消消乐障碍进阶章节、Candy Crush 复合障碍关。
   - **核心规则**：锁链、木箱、石块、蔓延块组合出现；玩家需先清理关键阻塞区域。
   - **建议关卡数**：24 关。
   - **首发必做**：障碍优先级提示、蔓延预警、失败原因标注为“障碍失控”。
   - **可延期内容**：障碍词缀系统（每周变体），Boss 型障碍。
   - **Excel 对应**：`Sheet17` 维护障碍图鉴与处理建议；`Sheet16` 顶部显示当前障碍威胁。

6. **特效连锁包：特殊块组合大师**
   - **定位**：强化高阶玩家爽点，鼓励主动做组合而非被动消除。
   - **原版参考**：Bejeweled 高级连锁、Candy Crush special combo 关。
   - **核心规则**：关卡目标直接要求触发特定组合（如“彩球+条纹”2 次）。
   - **建议关卡数**：14 关。
   - **首发必做**：组合任务追踪器、组合演示练习关、组合结算加权。
   - **可延期内容**：组合挑战排行榜、回放分享。
   - **Excel 对应**：`Sheet17` 的“组合实验室”可手动摆盘验证触发结果；`Sheet16` 显示组合计数器。

7. **限时冲分包：Blitz 与连击速刷**
   - **定位**：提供短局高压模式，增强重复游玩频率。
   - **原版参考**：Bejeweled Blitz、开心消消乐限时赛。
   - **核心规则**：60/120 秒固定倒计时，分数和连击为唯一目标。
   - **建议关卡数**：3 档时长 + 6 张主题盘。
   - **首发必做**：倒计时高优先级 HUD、连击中断反馈、最佳分数记录。
   - **可延期内容**：每周主题地图、限时增益词缀。
   - **Excel 对应**：`Sheet16` 模式切换卡新增 Blitz；状态栏实时显示剩余秒数。

8. **每日挑战与赛季回流包**
   - **定位**：构建长期回流，不依赖单次主线更新。
   - **原版参考**：Candy Crush Daily Challenge、移动端赛季活动。
   - **核心规则**：每日随机模板（目标+障碍+步数）与周任务组合；按完成度发放段位积分。
   - **建议关卡数**：首发 30 个模板轮换。
   - **首发必做**：每日任务入口、完成状态存档、最近 7 天结果可回看。
   - **可延期内容**：赛季排行榜、好友比拼、挑战回放。
   - **Excel 对应**：`Sheet1` Hub 显示今日挑战；`Sheet16` 直达每日关；`Sheet17` 展示本周障碍词缀。

## 6. HUD、结算、进度记录、练习与图鉴设计

### 6.1 HUD（不遮挡棋盘，最多两行信息）

- 第一行：模式、关卡名、目标摘要、剩余步数/时间。
- 第二行：分数、连锁段数、特殊块任务进度、障碍威胁。
- 右侧操作：暂停、重开、加速动画（仅表现层，不影响逻辑）。

### 6.2 结算面板

- 胜利：星级、分数明细、连锁峰值、关键目标完成率、推荐下一包。
- 失败：未完成目标清单、失败主因（步数不足/障碍失控/时间耗尽）、建议策略。
- 行为出口：`重开本局` / `返回选关` / `继续训练`。

### 6.3 进度记录（建议字段）

- `completedLevelIds`
- `levelStars`
- `bestScoreByLevel`
- `bestBlitzScore`
- `dailyChallengeHistory`
- `specialComboStats`（触发次数与最高连锁）
- `lastFailureReason`

### 6.4 练习模式

- 特殊块训练：直接给固定盘面，练“4 连/5 连/TL 形”制造。
- 组合训练：指定在 N 步内触发某组合。
- 障碍训练：单障碍与复合障碍拆解练习。
- 练习结果不扣体力、不影响主线步数资源。

### 6.5 图鉴/说明页（Sheet17）

- 块类型图鉴：普通色块、条纹、包装、彩球。
- 障碍图鉴：效果、击破方式、优先级建议。
- 目标图鉴：每种关卡目标的判定时机与计数方式。
- 组合手册：任意两种特殊块组合后的结果与范围示意。

## 7. 技术实现边界（按现有 2D 浏览器架构）

### 7.1 架构原则（来自 web-game-foundations）

- **状态与渲染分离**：规则在 simulation 层，UI 只消费状态并发动作。
- **输入动作统一**：`swap`, `pause`, `restart`, `toggle-speed`。
- **可存档状态序列化**：仅保存逻辑状态，不保存渲染对象。
- **DOM HUD + 棋盘渲染解耦**：棋盘可用 DOM/CSS Grid（首发），HUD/菜单保持 DOM 层。

### 7.2 模块拆分建议

- `src/features/match3/match3Types.ts`
- `src/features/match3/match3LevelCatalog.ts`
- `src/features/match3/match3BoardState.ts`
- `src/features/match3/match3ProgressStorage.ts`
- `src/components/match3/Match3Sheet.tsx`
- `src/components/match3/Match3Board.tsx`
- `src/components/match3/Match3Hud.tsx`
- `src/components/match3/Match3ResultPanel.tsx`
- `src/components/match3/Match3LabSheet.tsx`
- `src/styles/match3.css`

### 7.3 关键状态机

- `setup` -> `playing` -> `resolving` -> `won/lost`。
- `resolving` 子阶段：`markMatches` -> `clear` -> `drop` -> `spawn` -> `checkCascade`。
- 输入只在 `playing` 可用，`resolving` 期间锁定，避免竞态。

### 7.4 关卡数据化边界

- 关卡定义必须纯数据（JSON/TS 对象），禁止把关卡特例写进组件。
- 障碍、目标、掉落规则全部走 catalog 配置。
- 每个内容包对应独立 level group，便于后续增量上线。

### 7.5 测试基线

- 单元测试：
  - 合法/非法交换判定。
  - 匹配扫描准确性（含交叉群组）。
  - 掉落与补牌稳定性。
  - 特殊块生成与组合结果。
  - 洗牌后可行动作保证。
- 回归测试：
  - 关卡胜负判定。
  - 连锁倍率积分。
  - 进度存储与读取。

### 7.6 性能与可维护性

- 首发棋盘规模 8x8，DOM 渲染可控。
- 动画与逻辑解耦：逻辑帧先结算，动画通过事件队列播放。
- 长期扩展时，若障碍动画显著增多，可引入棋盘 Canvas 层，但 HUD/菜单继续 DOM。

## 8. 交付顺序建议（首发到扩展）

1. 先交付“基础三消+步数目标”双模式，保证核心 loop 稳定。
2. 再接入“果冻清理+下落收集”，完成主线可玩骨架。
3. 然后补“障碍拆解+特效连锁”提升策略深度。
4. 最后上线“限时冲分+每日挑战”做长期回流。

以上顺序能保证每一阶段都可发布、可测试、可扩展，不会出现“功能很多但没有完整可玩线”的交付风险。
