# Match3 模块 P0/P1/P2 详细扩展方案

- 日期：2026-04-15
- 适用模块：`Sheet16 / Sheet17`
- 目标：基于当前仓库内已落地的 Match3 骨架，补齐玩法完整度、内容包厚度和长线产品层，让开发可以直接按层拆任务推进
- 约束：只沿用当前仓库已有的 `状态层 + UI 壳层 + level catalog + progress storage + lab/practice` 架构，不重写模块边界

---

## 0. 当前仓库基线与设计原则

### 0.1 已有基线

当前 Match3 已经不是空壳，现状可以概括为：

1. **状态层已成立**  
   已有 `match3Types.ts`、`match3BoardState.ts`，支持基础棋盘、交换、扫描、掉落、连锁、4 类特殊块、基础障碍、传送口、蔓延块、胜负判定。
2. **内容层已起步**  
   已有 `match3LevelCatalog.ts`，当前只有 3 个正式关卡包：`新手入门包 / 步数策略包 / 果冻清理包`，共 50 关。
3. **UI 壳层已接通**  
   `Match3Sheet.tsx` 已能完成选关、开局、HUD、结算的基础闭环，`Match3Hud.tsx` 与 `Match3ResultPanel.tsx` 已有稳定结构。
4. **实验室与练习层有骨架**  
   `Match3LabSheet.tsx` 已有图鉴、目标、组合、练习标签页；`Match3PracticeModule.tsx` 已有部分练习逻辑，但未和 `Sheet17` 真正打通，且练习配置不完整。
5. **存档层可支撑主线关卡**  
   `match3ProgressStorage.ts` 已覆盖通关、星级、最佳分数、关卡摘要，但只适合“逐关 Adventure”记录，还不够支撑 Blitz、每日挑战、长线专项训练。

### 0.2 当前差口

当前真正卡住 Match3 从“可玩骨架”升级成“可持续模块”的，不是棋盘基础循环，而是下面 5 个缺口：

| 维度 | 当前状态 | 主要缺口 |
| --- | --- | --- |
| 关卡结构 | 只有 3 包 50 关 | 缺掉落收集、障碍压制、特效组合、限时冲分、每日挑战内容线 |
| 模式层 | 本质仍只有主线闯关 | `Blitz / Puzzle / Daily / 真 Practice` 没有独立入口和数据结构 |
| 机制层 | 4 类特殊块和部分障碍已可用 | 高阶组合要补齐，掉落目标仍未正式落地，现有目标系统对模式目标支持不够 |
| UI 层 | Sheet16/17 已有基础分工 | `继续训练` 语义不清，Sheet17 练习是假入口，HUD 还没有模式化信息密度 |
| 存档/长线 | 只有逐关进度 | 缺 `bestBlitzScore / comboStats / dailyChallengeHistory / failureReason` 等元进度 |

### 0.3 本方案的三条硬原则

1. **先把现有四类特殊块做满，再考虑新增基础特殊块类型**  
   当前 `Match3SpecialType` 只有 `striped-h / striped-v / wrapped / colorBomb`，P0/P1 先补齐组合矩阵、预置和目标，不在这一轮引入第五种基础特殊块。
2. **继续以内容包和关卡脚本为中心推进**  
   不把特殊规则硬塞进 `Match3Sheet.tsx`，所有模式和章节差异都回到 `match3LevelCatalog.ts` 对应的数据层。
3. **Sheet16 只负责“进入战斗与打完一局”，Sheet17 只负责“理解规则与专项训练”**  
   不把实验室重新塞回主战斗页，保持 PvZ / Tetris / Zuma 已经形成的多 Sheet 职责方式。

## 2. P0 详细设计：把核心玩法做完整

### 2.1 P0 目标

P0 不追求一下子把 8 包全部做完，而是先让 Match3 满足“经典三消该有的完整感”：

1. 现有 4 类特殊块的高阶组合全部真实可玩。
2. `dropCollect` 从类型定义变成正式关卡线。
3. `Sheet17` 练习不是说明卡，而是真模块。
4. `Sheet16` 的 HUD、结算、键盘语义与玩法一致。

### 2.2 P0 交付项

#### P0-A 高阶组合补齐

本批必须完成以下组合，并让它们能进入关卡目标与练习：

| 组合 | 当前定位 | P0 处理 |
| --- | --- | --- |
| 条纹 + 条纹 | 已有概念 | 保持，补进练习与组合目标 |
| 条纹 + 包装 | 已有概念 | 保持，补进 HUD 任务追踪 |
| 包装 + 包装 | 已有概念 | 保持，补进结算统计 |
| 彩球 + 普通块 | 已有概念 | 保持，补进专项练习 |
| 彩球 + 特殊块 | 需要正式收口 | 列为 P0 必做，作为高阶组合主目标 |
| 彩球 + 彩球 | 应视为终极组合 | 明确归入 P0，不新增类型，只做“全屏清场 + 障碍压制” |

#### P0-B 掉落目标正式落地

当前目标类型里已有 `dropCollect` 标签，但内容和状态都还没形成真正闭环。P0 需要把它做成正式第四包首发：

1. 新增掉落物实体表现
2. 新增底部出口判定
3. 掉落物支持通过传送口
4. 掉落物不可参与普通消除，只作为“被护送目标”
5. 结算要能明确告诉玩家“还差几个掉落物”

#### P0-C 练习闭环打通

当前 `Match3LabSheet` 的练习入口仍显示“功能开发中”，`Match3PracticeModule` 也缺障碍练习配置。P0 需要做到：

1. `Sheet17 -> 练习模式 -> 真模块`
2. `结算页 -> 继续训练 -> 对应专项`
3. 补齐缺失的 `obstacle-single / obstacle-complex`
4. 练习结果单独统计，不污染主线星级

#### P0-D 键盘与交互收口

当前键盘流程已经有部分支持，但仍偏“壳层键位”，不是标准三消手感。P0 定义如下：

1. 箭头键移动焦点
2. `Enter / Space` 选中当前格
3. 再次箭头决定交换方向
4. `Esc` 只在对局内负责暂停/退出弹层，不与失败结算冲突
5. 非法交换给出明确回弹和提示

### 2.3 P0 内容包补充

P0 结束时，Adventure 主线应从当前 3 包扩到 4 包：

| 包 | 当前状态 | P0 后状态 |
| --- | --- | --- |
| 包 1 新手入门包 | 已有 12 关 | 保持结构，补结业考试和组合目标可视化 |
| 包 2 步数策略包 | 已有 20 关 | 保持结构，补失败原因更细颗粒度 |
| 包 3 果冻清理包 | 已有 18 关 | 保持结构，补覆盖层/障碍复合提示 |
| 包 4 掉落收集包 | 未落地 | P0 新增 18 关，成为首个“路径规划型”内容线 |

## 3. P1 详细设计：把内容层做厚

### 3.1 P1 目标

P1 的任务是让 Match3 不再停留在“只有主线基础关卡”，而成为有清晰层级的内容模块：

1. Adventure 扩到 7 个正式内容包。
2. `Blitz` 与 `Puzzle` 成为真实可选模式。
3. 关卡脚本具备模板、模式、权重、预置特殊块能力。

### 3.2 P1 新增包与模式

#### 新增正式内容包

1. 障碍压制包
2. 特效连锁包
3. 限时冲分包

#### 新增模式

| 模式 | 定位 | 放置位置 |
| --- | --- | --- |
| Adventure | 主线逐关推进 | Sheet16 默认主模式 |
| Blitz | 60/90/120 秒冲分 | Sheet16 模式标签第二项 |
| Puzzle | 固定盘解题 | Sheet16 模式标签第三项 |
| Practice | 专项训练 | Sheet17 主入口，Sheet16 结算可回流 |

### 3.3 P1 关卡脚本补强

P1 建议把现有 `Match3LevelScript` 扩成“关卡脚本 + 模式脚本”双层：

- `modeId`
- `chapterId`
- `chapterName`
- `boardTemplateId`
- `spawnProfileId`
- `colorWeights`
- `prebuiltSpecials`
- `dropTargets`
- `mutators`

这样可以避免把 `Blitz / Puzzle / Daily` 继续硬塞成普通关卡。

## 4. P2 详细设计：补长线与产品层

### 4.1 P2 目标

P2 不再以“再做多少关”为核心，而是让 Match3 形成回流：

1. 每日挑战与周常词缀成立
2. 专项训练与失败归因成立
3. 存档里有真正的长期目标数据

### 4.2 P2 新增结构

| 层级 | 内容 |
| --- | --- |
| 日常回流 | 每日挑战、周常任务、赛季节点 |
| 技术成长 | 组合专项、障碍专项、限步优化专项 |
| 元进度 | 连锁纪录、最佳 Blitz、每包完成度、失败原因分布 |
| 实验室升级 | 规则词缀、组合演示、训练推荐直达 |

### 4.3 P2 核心原则

1. **不把每日挑战和主线关卡混在一个存档字段里**  
   主线是永久进度，每日挑战是轮换记录。
2. **不引入复杂付费养成**  
   元进度只服务于“推荐下一个挑战”和“让玩家知道自己进步了”。
3. **训练回流必须和失败原因绑定**  
   例如连续 3 次失败都因“掉落物卡中段”，则推荐进入“掉落路径专项”。

## 5. 详细关卡结构：8 个正式内容包 / 章节拆解

本节是开发执行的核心。每个包都明确：章节目标、节奏、核心机制、实现备注。

### 5.1 包 1：新手入门包（12 关，保留现有包，P0 收口）

- 定位：建立交换、三消、基础特殊块直觉
- 目标：让玩家在最短时间内理解“合法交换、4 连、5 连、T/L 型”
- 节奏：教学明显、失败成本低、每 4 关一个小考试

| 章节 | 关卡 | 章节目标 | 节奏 | 核心机制 |
| --- | --- | --- | --- | --- |
| A 交换基础 | 1-4 | 建立三消与非法交换回弹认知 | 低压，3-4 色盘面 | 合法交换、基础计分、简单连锁 |
| B 特殊块认识 | 5-8 | 学会四连条纹、五连彩球、T/L 包装 | 中低压，开始要看形状 | 条纹、彩球、包装块生成 |
| C 新手毕业 | 9-12 | 用简单目标组合做完整一局 | 低到中压，强调复盘 | 连锁、特殊块触发、结算星级 |

实现备注：

- 继续使用当前 `beginner` pack 数据。
- P0 只补“关卡描述、考试关说明、组合目标提示”，不大改结构。

### 5.2 包 2：步数策略包（20 关，保留现有包，P0 收口）

- 定位：把“看得见消除”升级为“规划每一步”
- 目标：让玩家理解分数、颜色收集、特殊块效率的关系
- 节奏：步数逐步压缩，颜色目标与分数目标交替

| 章节 | 关卡 | 章节目标 | 节奏 | 核心机制 |
| --- | --- | --- | --- | --- |
| A 单目标节省步数 | 1-6 | 在低步数下完成单目标 | 中压 | 分数关、单色收集、基础步效 |
| B 双色与组合效率 | 7-13 | 学会在多目标下选择优先级 | 中压到中高压 | 双色收集、组合效率、局部解场 |
| C 结业考试 | 14-20 | 在低容错下完成复合目标 | 中高压 | 颜色 + 分数复合目标、特殊块优先级 |

实现备注：

- 保持当前 `move-strategy` 20 关框架。
- P0 重点补失败原因分支和“浪费步数”提示。

### 5.3 包 3：果冻清理包（18 关，保留现有包，P0 收口）

- 定位：把目标从“收集数值”切到“区域清理”
- 目标：教会玩家主动围绕障碍区域做解场
- 节奏：从单层覆盖过渡到双层覆盖与局部封盘

| 章节 | 关卡 | 章节目标 | 节奏 | 核心机制 |
| --- | --- | --- | --- | --- |
| A 单层覆盖 | 1-6 | 理解覆盖层需要在目标区域触发消除 | 中压 | `clearOverlay`、局部清理 |
| B 双层覆盖 | 7-12 | 学会用条纹/包装快速处理双层区域 | 中高压 | 双层冰冻、区域优先级 |
| C 覆盖考试 | 13-18 | 在有限步数下清完大面积覆盖 | 高压 | 连锁辅助、特殊块解区域 |

实现备注：

- 保持当前 `jelly-clear` pack，补“区域高亮”和“剩余覆盖格”展示。
- 不在 P0 里新增复杂新障碍，先把覆盖层关做扎实。

### 5.4 包 4：掉落收集包（18 关，P0 新增）

- 定位：让 Match3 从“纯消除”转向“路径规划”
- 目标：建立掉落物、出口、列控制与传送口的使用方法
- 节奏：先单物件直降，再多出口，再障碍封路

| 章节 | 关卡 | 章节目标 | 节奏 | 核心机制 |
| --- | --- | --- | --- | --- |
| A 单物件护送 | 1-6 | 学会把单个掉落物送到底部出口 | 中压 | `dropCollect`、底部出口、列清理 |
| B 多物件分流 | 7-12 | 学会处理双掉落物与多出口 | 中高压 | 多出口、掉落顺序、路径优先级 |
| C 封路考试 | 13-18 | 在障碍和传送干扰下完成掉落目标 | 高压 | 木箱/石块封路、传送口改道、限步护送 |

实现备注：

- 这是 P0 最关键的新内容包。
- 首发建议 18 关，和果冻包体量接近，足够建立独立内容线。

### 5.5 包 5：障碍压制包（24 关，P1 新增）

- 定位：让障碍真正成为 Match3 的中后期主压力
- 目标：建立“先解场，再冲目标”的思考方式
- 节奏：从锁链木箱到石块、蔓延、传送复合封盘

| 章节 | 关卡 | 章节目标 | 节奏 | 核心机制 |
| --- | --- | --- | --- | --- |
| A 锁链木箱 | 1-8 | 学会优先清关键阻塞点 | 中高压 | 锁链、木箱、邻接破坏 |
| B 石块与传送 | 9-16 | 学会远程清理与路径绕行 | 高压 | 石块、传送口、远程解场 |
| C 蔓延考试 | 17-24 | 学会在蔓延失控前抢节奏 | 高压到很高压 | `spreader`、威胁压制、障碍复合 |

实现备注：

- 该包上线后，`Match3Hud` 的威胁区才真正有长期价值。
- 也是 `Sheet17` 障碍图鉴最需要服务主玩法的阶段。

### 5.6 包 6：特效连锁包（16 关，P1 新增）

- 定位：把高阶玩家爽点做成正式内容线
- 目标：不靠运气触发，而是让玩家主动制造并完成指定组合
- 节奏：先指定生成，再指定组合，再连锁考试

| 章节 | 关卡 | 章节目标 | 节奏 | 核心机制 |
| --- | --- | --- | --- | --- |
| A 生成专项 | 1-5 | 定向制造条纹/彩球/包装 | 中压 | 特殊块生成目标、预置提示 |
| B 组合专项 | 6-10 | 触发指定组合达成目标 | 高压 | 条纹+包装、彩球+特殊块、包装+包装 |
| C 爽点考试 | 11-16 | 用组合完成大面积清场和高连锁 | 高压 | 组合链、爆发解场、连锁峰值 |

实现备注：

- 该包不新增新砖种，只把现有四类特殊块做出高级内容价值。
- 可同时作为 `Sheet17` 组合训练的主回流池。

### 5.7 包 7：限时冲分包（12 张主题图 + 3 档时长，P1 新增）

- 定位：短局复玩模式
- 目标：服务想要“爽一局”的玩家，不要求逐关推进
- 节奏：60 秒进门、90 秒进阶、120 秒耐力

| 章节 | 配置 | 章节目标 | 节奏 | 核心机制 |
| --- | --- | --- | --- | --- |
| A Blitz-60 | 4 张图 | 学会在短倒计时里最大化收益 | 极高压 | 快速找 4 连、彩球冲分、不中断连击 |
| B Blitz-90 | 4 张图 | 平衡分数与清场稳定性 | 高压 | 连击续航、颜色权重、节奏保持 |
| C Blitz-120 | 4 张图 | 追求高分与长连锁纪录 | 中高压 | 大爆发、Combo 记录、耐力规划 |

实现备注：

- 不必按主线“逐关解锁”，而是按最好成绩记录。
- `maxTimeMs` 现有字段可直接接入这个模式。

### 5.8 包 8：每日挑战与赛季回流包（P2 新增）

- 定位：长期回流与内容刷新层
- 目标：不改核心战斗逻辑，只换模板、词缀、目标组合
- 节奏：每日一关、每周一词缀、赛季一批挑战

| 章节 | 配置 | 章节目标 | 节奏 | 核心机制 |

实现备注：

- 这包本质是“模板层”，不是独立棋盘机制层。
- 应该复用前 7 包的模板与规则组合，而不是再造一套玩法。

## 6. 玩法机制补充方案

### 6.1 高阶特殊块与组合机制

本方案不推荐在 P0/P1 引入新的基础特殊块，而是把现有四类做成真正完整的高级玩法。

#### 组合矩阵

| 组合 | 作用 | 关卡使用阶段 | 备注 |
| --- | --- | --- | --- |
| 条纹 + 条纹 | 十字清除 | P0 | 新手毕业和组合练习都要覆盖 |
| 条纹 + 包装 | 三行三列爆破 | P0 | 作为高阶障碍破局核心 |
| 包装 + 包装 | 大范围双爆 | P0 | 作为中后期“破封盘”核心 |
| 彩球 + 普通块 | 清屏该色 | P0 | 颜色收集和分数关基础爽点 |
| 彩球 + 特殊块 | 全色转化并触发 | P0 | 三消高阶爽点主轴 |
| 彩球 + 彩球 | 全屏清场 | P0 | 应定义为“终极组合”，但控制关卡频率 |

#### 关卡目标层的补充

P1 开始允许以下组合型目标进入关卡脚本：

- `createSpecial: striped / wrapped / colorBomb`
- `triggerCombo: striped-striped`
- `triggerCombo: striped-wrapped`
- `triggerCombo: wrapped-wrapped`
- `triggerCombo: colorBomb-special`
- `maxCascadeTarget`

#### 不建议本轮做的事

1. 不新增第 5 种基础特殊块
2. 不做复杂能量条技能
3. 不把特殊块升级成 RPG 装备式养成

### 6.2 掉落目标机制

掉落目标建议定义为“非匹配实体 + 出口判定”的单独层，而不是把掉落物强行塞成普通色块。

#### 规则定义

1. 掉落物占据一个格位，但不参与普通三消
2. 掉落物只能靠下方清空继续下落
3. 掉落物可经过 `portalIn -> portalOut`
4. 掉落物进入底部出口后从棋盘移除并计数
5. 掉落物不能被普通特殊块清除，只能被“护送到出口”完成

#### 首发掉落物类型

| 类型 | 用途 | 首发阶段 |
| --- | --- | --- |
| 星星文件夹 | 教学型掉落物 | P0 |
| 审批印章 | 多出口关 | P0 |
| 公式芯片 | 传送口混合关 | P1 |

### 6.3 模式层

建议把 Match3 的模式从“关卡包切换”升级成“模式 + 包”的双层结构：

| 模式 | 数据来源 | 胜负方式 | 记录方式 |
| --- | --- | --- | --- |
| Adventure | 逐关脚本 | 完成所有目标 | 星级 / 最佳分 / 最佳步数 |
| Blitz | 地图模板 + 时间脚本 | 时间结束结算 | 最佳分 / 最长连击 |
| Puzzle | 固定盘脚本 | 在限定步内达成解题目标 | 通关 / 最优步数 |
| Practice | 专项练习脚本 | 达成训练目标即可 | 通过率 / 最近成绩 |
| Daily | 模板池 + seed | 当天限定目标 | 当日成绩 / 最近 7 天 |

### 6.4 练习层

练习层必须从“说明页按钮”升级成真正可回流的训练层。

## 7. 实现拆解

P2 时保留旧字段兼容读取，再补一次 `validateAndRepairProgress` 迁移，不清空旧存档。

## 10. 最终决策摘要

1. **P0 不新增新的基础特殊块类型，先把现有四类做满。**
2. **主线先从 3 包扩到 4 包，第四包必须是掉落收集包。**
3. **P1 再把内容扩到 7 包，并正式引入 Blitz 与 Puzzle。**
4. **P2 再做每日挑战、元进度和实验室长期回流。**
5. **Sheet16 负责进入一局并完成一局，Sheet17 负责理解与训练。**
6. **掉落目标必须做成独立实体层，不要把掉落物伪装成普通色块。**
7. **练习记录必须和主线进度分离。**
8. **关卡差异继续优先走 `level catalog` 数据化，不把特例塞回组件。**

这套方案的核心不是“再多写一些关”，而是让 Match3 从当前的基础闭环，升级成一个和 PvZ、Tetris、Zuma 一样可持续扩展的正式模块。
