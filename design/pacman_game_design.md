# 吃豆人（Pac-Man）Excel 游戏详细设计

- 日期：2026-04-14
- 项目：`Eexcl_dawa`
- 设计目标：在现有 Excel 风格游戏中心中高还原 Pac-Man 原版体验，并预留章节化扩展能力
- 设计阶段：实施前详细设计（可直接拆解任务）

---

## 1. 设计目标与原版体验支柱

### 1.1 总体目标

在不脱离工作簿壳层的前提下，做出一套“规则确定、路线可学、风险可控、节奏不断加压”的 Pac-Man 正式模块。  
优先还原原版核心循环，再在同一套状态机上增加训练和挑战内容。

### 1.2 原版体验支柱

1. **固定迷宫 + 路线学习**
   玩家通过重复跑图形成“安全线”和“高收益线”，而非随机反应。
2. **四鬼差异 AI**
   Blinky / Pinky / Inky / Clyde 不是同一追踪算法，压迫来自协同包夹。
3. **模式节奏切换**
   Scatter 与 Chase 的时序切换决定节奏脉冲，切换时鬼反向是关键可读反馈。
4. **能量豆高风险收益**
   吃能量豆后的“短窗口反打”是分数与生存的核心权衡。
5. **清图胜利条件**
   以清空豆子推进关卡，不是纯生存倒计时。
6. **可记忆的难度曲线**
   速度、惊吓时长、Energizer 收益、水果奖励逐关变化且可被学习。

## 2. 适配当前游戏中心与 Sheet 职责建议

> 本文中的 Sheet 编号按当前新增三套经典模块的统一接入方案固定为 `Sheet12 / Sheet13`。若后续整体表结构调整，应以总览蓝图为准同步改号。

### 2.1 为什么适合本项目

- 当前项目已有 Snake / Tetris / PvZ 的“状态层 + UI 壳层”模式，Pac-Man 同样适合以离散状态 + 高频 tick 驱动。
- 迷宫与豆子天然可映射为 Excel 栅格语义（单元格通路、阻塞、奖励点）。
- 单局时长可控（2~6 分钟），与现有模块切换节奏一致。
- 高可扩展：同一运行时可扩展训练关、速通关、无伤关、镜像关。

### 2.2 Sheet 职责建议

- `Sheet12（首发必做）`：选关、模式设置、正式对局、HUD、结算。
- `Sheet13（可延期）`：鬼魂图鉴、模式说明、路径练习与回放诊断。
- `Hub`：展示 Pac-Man 的最好分、最高关卡、最快清图、无伤记录。

## 3. 核心 Loop 与系统规则

### 3.1 胜负条件

- **关卡胜利**：当前迷宫所有普通豆清空。
- **关卡失败**：生命值归零。
- **章节通关**：按章节定义完成若干关或达成目标（如 5 关连续通关）。

### 3.2 输入模型

- 方向输入：`Arrow / WASD`。
- 动作输入：`Pause`、`Restart`、`Exit`。
- 输入策略：缓存“下一方向”，在进入新格前判定可转向时执行，保证手感接近原版。

### 3.3 核心循环

1. 读取输入，更新 Pac-Man 方向与位移。
2. 处理豆子、能量豆、水果拾取与得分。
3. 更新四鬼状态机（House / Scatter / Chase / Frightened / Eaten / Respawn）。
4. 进行碰撞判定（Pac-Man vs Ghost / Wall / Tunnel）。
5. 结算当前帧事件（加分、掉命、清图、进关、音效触发）。
6. 推送 HUD 与状态栏摘要。

### 3.4 地图、豆子、能量豆、传送门

- 地图单元类型：`wall / path / pellet / energizer / ghostHouse / tunnel / fruitSpawn`。
- 标准迷宫：首发至少包含原版经典迷宫 1 张，并支持后续多迷宫目录。
- 豆子总量：按地图定义（经典图约 240 粒），不写死在代码常量。
- 能量豆：固定在 4 个角位，触发 Frightened 全局状态窗口。
- 传送门：左右通道联通，鬼在 tunnel 中应用速度修正。

### 3.5 水果系统

- 每关按豆子剩余阈值触发水果（建议 2 次触发窗口）。
- 水果序列使用原版风格进阶（樱桃、草莓、橙子、苹果、甜瓜、银河舰、铃铛、钥匙）。
- 水果出现时间有限，提供“路线压缩决策”。

### 3.6 鬼魂 AI 与状态机

#### 四鬼目标逻辑（高还原）

- **Blinky（红）**：直接追踪 Pac-Man 当前格目标。
- **Pinky（粉）**：瞄准 Pac-Man 前方若干格（含原版“向上时偏移异常”可配置开关）。
- **Inky（青）**：以 Blinky 与 Pac-Man 前方点构造向量目标，形成夹击。
- **Clyde（橙）**：远距离追击，近距离回角落，形成节奏断点。

#### 全局模式时序

- 模式按“Scatter/Chase 周期表”驱动，不用纯随机切换。
- 切换时鬼执行一次反向，保证玩家可读。
- Frightened 结束后恢复原模式节奏，不重置整局时序。

#### 鬼屋放行机制

- 采用“豆子计数 + 时间兜底”的混合放行规则。
- 不同鬼放行门槛独立配置，确保开局压力可控且有学习价值。

## 4. 原版节奏、关卡与难度曲线（还原策略）

### 4.1 速度与时序表驱动

首发使用 `levelTuningTable` 做参数表，不把速度和惊吓时长写死在逻辑分支中。  
关键字段建议：

- `pacmanSpeed`
- `ghostSpeed`
- `tunnelGhostSpeedMultiplier`
- `frightenedDurationMs`
- `frightenedBlinkMs`
- `scatterChaseSchedule`
- `fruitValueSet`

### 4.2 关卡阶段建议（对应原版学习曲线）

1. `Level 1-2`：惊吓窗口长，允许学习鬼行为。
2. `Level 3-5`：追击速度提升，强制优化路线。
3. `Level 6-9`：惊吓窗口变短，反打价值变成“窗口技巧”。
4. `Level 10-16`：更强调避让与路径预判。
5. `Level 17+`：接近原版高压状态，主要靠路线稳定性过图。

### 4.3 还原优先级

- P0：鬼目标逻辑、模式时序、清图胜利、能量豆反打、生命与重生。
- P1：水果触发序列、鬼屋精细放行、鬼速度细分（含 tunnel 修正）。
- P2：额外迷宫、镜像变体、训练回放、挑战脚本编辑器。

## 5. 正式内容包（8 个）

## 5.1 经典街机包（Classic Arcade）

- 定位：完整交付 Pac-Man 核心体验，作为默认入口。
- 原版参考：1980 原版 Pac-Man 主循环。
- 核心规则：单迷宫、3 命、清图进关、四鬼差异 AI、能量豆反打、水果奖励。
- 建议关卡数：21+ 循环关（以参数表递进）。
- 首发必做：完整 Scatter/Chase/Frightened 状态机，HUD 基础信息，结算与记录。
- 可延期内容：更精细的逐关时序拟合、原版细碎 bug 兼容开关。
- Excel 对应：`Sheet12` 默认模式入口，`StatusBar` 显示关卡/命数/分数。

## 5.2 路线教学包（Route Training）

- 定位：让新玩家学会“安全路线 + 收豆顺序 + 能量豆时机”。
- 原版参考：街机背板路线教学与玩家社区经典跑图。
- 核心规则：低速版节奏，显示推荐路径点，允许阶段复盘。
- 建议关卡数：10 关（按路口决策点拆分）。
- 首发必做：练习提示层、分段 checkpoint、失败后快速重试。
- 可延期内容：动态评分（偏离推荐线扣分）。
- Excel 对应：`Sheet13` 训练页，`Sheet12` 可一键跳转练习。

## 5.3 四鬼认知包（Ghost Behavior Lab）

- 定位：把四鬼差异从“感觉”变成可观察规则。
- 原版参考：社区 Ghost AI 研究图。
- 核心规则：每关只强化一个鬼或一种组合包夹，提供热力覆盖可视化。
- 建议关卡数：12 关（单鬼 4 + 双鬼 4 + 四鬼 4）。
- 首发必做：鬼目标点可视化调试层（仅训练模式）。
- 可延期内容：轨迹回放与错误原因自动标注。
- Excel 对应：`Sheet13` 的“鬼魂实验室”子页。

## 5.4 水果冲分包（Fruit Score Rush）

- 定位：强调风险换收益的高分路线。
- 原版参考：原版水果刷新与高分跑法。
- 核心规则：水果触发更频繁但窗口更短，要求绕路抓高分。
- 建议关卡数：8 关。
- 首发必做：水果刷新倒计时提示、结算显示水果得分占比。
- 可延期内容：水果组合连击加成。
- Excel 对应：`Sheet12` 模式卡“冲分赛”，Hub 增加“最高水果连取”。

## 5.5 限命挑战包（One-Life Challenge）

- 定位：面向熟练玩家的高压稳定性测试。
- 原版参考：街机一命通关挑战。
- 核心规则：全程 1 命，无复活；失败即重开章节。
- 建议关卡数：5 章（每章 3~5 关）。
- 首发必做：章节断点记录、失败原因清单、重开极速流程。
- 可延期内容：段位徽章与赛季挑战。
- Excel 对应：`Sheet12` 独立挑战入口，`StatsPanel` 增加“一命最远关”。

## 5.6 速通竞赛包（Time Attack）

- 定位：把清图效率变成主目标，减少拖局刷分。
- 原版参考：社区速通与最短路线挑战。
- 核心规则：按“限定时间内清图”或“最短时间清图”结算。
- 建议关卡数：15 关（短图 5 / 中图 5 / 高压图 5）。
- 首发必做：毫秒级计时、关内 split、结算 PB 对比。
- 可延期内容：录像导出与幽灵回放。
- Excel 对应：`Sheet12` 加速通标签，状态栏显示实时 split。

## 5.7 迷宫变体包（Maze Variants）

- 定位：在不改核心规则下扩大内容寿命。
- 原版参考：后续系列多迷宫变化。
- 核心规则：镜像图、窄道图、高 tunnel 权重图，豆子布局变化但 AI 规则不变。
- 建议关卡数：20 关（按地形主题分组）。
- 首发必做：地图目录与难度标签系统。
- 可延期内容：用户自定义迷宫导入校验器。
- Excel 对应：`Sheet12` 选图下拉 + 预览，`Sheet13` 提供地图解析说明。

## 5.8 无尽生存包（Endless Survival）

- 定位：长期回流内容，对接 Hub 长期记录。
- 原版参考：街机循环高关生存。
- 核心规则：关卡循环且参数持续加压，目标是尽量活更久、得分更高。
- 建议关卡数：无上限（按阶段显示）。
- 首发必做：阶段标记、压力摘要、失败前 30 秒事件记录。
- 可延期内容：周榜、好友比较、每日 modifiers。
- Excel 对应：`Hub` 增加 Pac-Man 生存榜，`Sheet12` 提供一键再来一局。

## 6. HUD、结算、进度记录、练习模式、图鉴/说明

### 6.1 HUD（战斗页）

必须字段：

- 当前分数
- 当前关卡
- 剩余生命
- 当前模式（Scatter/Chase/Frightened）
- 剩余豆子数
- 水果状态（未出 / 出现中 / 已拾取）

可选字段：

- 当前鬼速度倍率
- Frightened 剩余时间
- 当前关卡参数档位

### 6.2 结算页

- 胜利结算：用时、剩余命、豆子清空率、水果收益、吃鬼次数。
- 失败结算：死亡位置热区、最后 10 秒输入轨迹、建议改进点。
- 操作按钮：`重开本局`、`返回选关`、`进入练习`。

### 6.3 进度记录

建议新增 `pacmanProgressStorage`：

- `bestScore`
- `highestLevel`
- `bestClearTimeByMap`
- `bestOneLifeStage`
- `ghostsEatenTotal`
- `fruitsCollectedByType`
- `modeCompletion`

### 6.4 练习模式

- 路口转向练习（只练拐点判定）
- 能量豆反打练习（短窗口连吃鬼）
- 四鬼包夹逃生练习
- 指定段落复盘练习（从某时间点继续）

### 6.5 图鉴/说明页

- 四鬼行为说明（目标规则 + 反制建议）
- 水果价值与触发表
- 迷宫标签（tunnel 比例、死角密度、推荐路线类型）
- 输入与判定说明（缓存转向、碰撞优先级）

## 7. 技术实现边界（基于当前 2D 浏览器架构）

### 7.1 实现路径选择

本项目首发建议继续沿用现有 `React + 逻辑状态层` 架构，不强依赖 Phaser。  
原因：当前 Snake / Tetris / PvZ 已形成同类模块边界，复用成本最低，测试与状态持久化路径已成熟。

### 7.2 模块拆分建议

建议新增：

- `src/features/pacman/pacmanTypes.ts`
- `src/features/pacman/pacmanBoardState.ts`
- `src/features/pacman/pacmanAi.ts`
- `src/features/pacman/pacmanMapRegistry.ts`
- `src/features/pacman/pacmanLevelTuning.ts`
- `src/features/pacman/pacmanContent.ts`
- `src/features/pacman/pacmanSelectors.ts`
- `src/features/pacman/pacmanStorage.ts`

建议新增组件：

- `src/components/pacman/PacmanSheet.tsx`
- `src/components/pacman/PacmanBoard.tsx`
- `src/components/pacman/PacmanHud.tsx`
- `src/components/pacman/PacmanOverlay.tsx`
- `src/components/pacman/PacmanGuideSheet.tsx`（可延期）

### 7.3 Simulation 与 Rendering 边界

- Simulation：
  网格坐标、移动状态机、鬼 AI、碰撞、计分、关卡推进。
- Rendering：
  DOM/CSS 渲染迷宫、角色、豆子、UI 动画；不持有真状态。
- Tick：
  固定步进（建议 60Hz 逻辑步），渲染可跟随浏览器帧率。

### 7.4 数据与存档边界

- 仅存可序列化状态，不存 DOM 运行态。
- 存档分为：`settings`、`progress`、`lastRunStats`。
- 所有关卡参数走内容表，避免逻辑分支硬编码。

### 7.5 测试边界

首发最小测试集建议：

1. 路径阻挡与转向缓存判定。
2. 四鬼目标计算正确性（含 Pinky 特例开关）。
3. 模式时序切换与反向行为。
4. Frightened 触发/结束与吃鬼计分递增。
5. 清图进关与失败结算。
6. 水果触发阈值与超时消失。

## 8. 分阶段落地建议

### Phase A（可玩闭环）

- 单迷宫 + 经典模式 + 四鬼基础 AI + HUD + 结算 + 存档。

### Phase B（高还原）

- 完整模式时序表、速度表、水果表、鬼屋放行细节、训练入口。

### Phase C（扩展内容）

- 多迷宫包、速通包、一命包、无尽包、图鉴与复盘系统。

## 9. 交付判定标准（Done Definition）

- 玩家可在 `Sheet12` 完成“进局 -> 清图/失败 -> 结算 -> 重开/选关”的完整流程。
- 四鬼行为具备可识别差异，非同质追踪。
- 模式节奏变化可感知，能量豆窗口具有明确策略价值。
- 进度记录可追踪长期表现，不因刷新丢失关键历史。
- 新增模式包能在 `pacmanContent` 表驱动接入，无需改核心状态机。
