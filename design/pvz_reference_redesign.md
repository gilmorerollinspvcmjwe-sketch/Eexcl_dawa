# PvZ 参考原版重设计

- 日期：2026-04-12
- 模块：`Sheet7 / Sheet8 / Sheet9`
- 目标：参考原版《植物大战僵尸》的核心玩法，重新定义我们这个 Excel 版本应该保留什么、替换什么、怎么实现

---

## 1. 先说结论

我们这个版本的 PvZ，不能只做成“在格子上放植物打僵尸”。

它必须保留原版最核心的 5 个体验：

1. 资源经营
2. 卡组选择
3. 防线分路
4. 波次压迫
5. 章节扩展

如果没有这 5 个体验，它最多只是一个塔防雏形，不是 PvZ。

---

## 2. 原版 PvZ 的玩法核心

### 2.1 原版为什么成立

原版 PvZ 的魅力，不只是植物和僵尸有名字，而是它把几层策略压缩在了很容易理解的操作里：

- 先选什么植物
- 先种经济还是先种输出
- 哪一路先顶住
- 哪一波要预留资源
- 不同章节为什么打法完全不同

所以它的核心不是“摆塔”，而是：

**“我用有限资源组织一条能撑过整波压力的防线。”**

### 2.2 原版必须保留的支柱

我们版必须保留：

- 资源是有限且重要的
- 选卡决定打法
- 每一路都可能出事
- 僵尸种类会改变应对方式
- 不同章节真的应该有不同规则

我们版可以简化，但不能破坏：

- 不能让章节只是换文案
- 不能让卡组选择失去意义
- 不能让后期压力只靠堆血量

---

## 3. 我们版的现实约束

我们是 Excel 工作簿里的模块，不是完整大体量 PvZ 克隆。

固定约束有：

- 棋盘尺寸较小
- UI 空间有限
- 需要拆成多个 Sheet
- 玩家不适合一局打特别久
- 当前实现更偏“模块化合辑”而不是独立长篇单机

所以我们版 PvZ 应该偏向：

- 章节化短关卡
- 关卡规则差异清楚
- 图鉴和实验室承担说明与回流
- 主战斗页只保留最关键决策信息

---

## 4. 原版玩法到我们版的对应关系

| 原版要素 | 原版作用 | 我们版对应实现 |
| --- | --- | --- |
| 选卡 | 先做策略准备 | `Sheet7` setup 阶段 |
| 阳光经济 | 资源与节奏 | 阳光值、生产植物、前期决策 |
| 植物定位差异 | 打法多样性 | registry 中的能力角色和卡组池 |
| 分路防线 | 棋盘压力 | 5 路棋盘与每路局势 |
| 波次推进 | 持续压力 | 刷怪队列 / 章节脚本 |
| 章节切换 | 规则变化 | `Sheet9` 章节数据驱动 `Sheet7` |
| 图鉴 | 单位理解与回流 | `Sheet8` 图鉴页 |

关键点是：

**原版 PvZ 的强度，不在单位数量，而在“章节、卡组、经济、分路”四层是串起来的。**

---

## 5. 我们版应该怎么重做玩法层

### 5.1 核心循环重定义

正式版 PvZ 的循环应该是：

1. 选章节
2. 看本章规则和推荐卡组
3. 进入选卡
4. 开局先处理经济和首波压力
5. 中局根据僵尸类型修防线
6. 打完后回流到章节、图鉴和实验室

这意味着 PvZ 不能只停留在：

- 开局
- 种植物
- 杀僵尸
- 结束

而要形成完整“关前准备 -> 战斗 -> 复盘 -> 再选”的结构。

### 5.2 章节要怎么对应原版

#### 白天

对应原版的教学与基础资源节奏。

我们版应实现为：

- 自然资源宽松
- 基础僵尸为主
- 用来教会玩家阳光与输出平衡

#### 夜晚

对应原版的经济紧张和节奏变化。

我们版不一定要完整复刻蘑菇体系，但必须让玩家感受到：

- 资源节奏变了
- 不能再按白天打法照搬

#### 泳池 / 迷雾 / 屋顶

这些章节在我们版里不一定第一轮就全做完整地形特殊机制，但至少要做到：

- 默认卡组不同
- 僵尸组合不同
- 波次结构不同
- 章节目标和威胁不同

否则章节只是名字变化。

### 5.3 选卡系统要怎么对应原版

原版 PvZ 的选卡不是装饰，而是策略入口。

所以我们版选卡必须做到：

- 玩家知道自己为什么换卡
- 章节会影响选卡建议
- 默认卡组不是随便给，而是场景推荐

这也是为什么 `Sheet9` 和 `Sheet7` 必须联动。

## 6. 我们版应该怎么重做体验层

### 6.1 战前体验

PvZ 的成品感很大一部分来自战前准备。

我们版战前体验必须回答：

- 这是哪一章
- 这一章有什么威胁
- 默认卡组是什么思路
- 我可以自己换什么

如果这些信息不清楚，玩家就会把它当成“随便进一局的塔防页”。

### 6.2 战斗体验

原版 PvZ 的战斗不是忙乱点击，而是稳定判断。

我们版战斗体验应强化：

- 当前波次进度
- 当前阳光状态
- 哪一路最危险
- 当前选中的植物作用

但不能让 HUD 把棋盘盖住。

### 6.3 战后体验

PvZ 的回流价值比 Snake 和 Tetris 更重要。

我们版战后必须让玩家知道：

- 这章打过没
- 本局为什么赢 / 为什么输
- 下次该换什么卡
- 可不可以去图鉴和实验室看更细信息

---

## 7. 我们版对应实现原则

### 7.1 必须做的

- 章节真正驱动战斗开局
- 默认卡组和章节绑定
- 波次脚本和章节绑定
- 结算能回流到选卡 / 图鉴 / 实验室
- 图鉴和实验室不再只是说明页，而是主战斗辅助页

### 7.2 可以做但不要抢核心

- 更多植物和僵尸
- 更丰富的章节文案
- 单位短标签和角色包装
- 轻量章节推荐

### 7.3 暂时不要过早做太重的

- 大量特殊地图机制同时上线
- 过多章节一次性全开
- 复杂 meta 成长树
- 太重的剧情文本

---

## 8. 分阶段实现建议

### 第一阶段：原版结构对齐

目标：

- 章节影响战斗
- 选卡真正有意义
- 关卡波次和默认卡组成体系

### 第二阶段：章节差异成型

目标：

- 白天、夜晚、泳池、迷雾、屋顶打出不同体验
- 图鉴和实验室开始承担策略辅助作用

### 第三阶段：成品内容层

目标：

- 更完整单位规则
- 章节通关记录
- 推荐卡组
- 长期内容更新位

---

## 9. 最终设计判断

PvZ 在我们这个版本里的正确方向，不是先狂加单位，而是：

**先把原版的章节、卡组、经济、分路防线这几层重新串起来，再做内容扩张。**

也就是说：

- 原版提供策略骨架
- Excel 提供工作簿式承载方式
- 我们的版本负责把章节、图鉴、实验室拆成适合这个合辑的多 Sheet 结构

这才是正确重设计。
