# 贪吃蛇关卡与模式正式规划

- 日期：2026-04-12
- 模块：`Sheet10`
- 目标：在原版贪吃蛇多样模式框架上，结合文字链系统与 Excel 壳层的短局节奏，制定可执行的关卡包、地图尺寸与难度梯度，使得每种玩法既有原版支柱，又能在当前项目里找出差异化。

---

## 1. 原版模式与关卡谱系（经典 / 限时 / 障碍 / 任务 / 字符链扩展）

原版贪吃蛇在不同平台和时代常见的模式与关卡设计可以按以下维度展开：

1. **经典（标准）模式**：最基础的 `单食物、无限生长`。每颗食物的价值与风险一致，随着蛇身拉长，走位空间被不断压缩，玩家通过控制节奏和路线来延长存活时间并追求高分。地图通常是单一区域，边界固定，偶尔出现平滑角落或障碍变种。

2. **限时冲分**：常见于手机版/消磨类变种，以固定时间（如60s）挑战最多的食物或分数。玩法核心转向“每一秒都要创造价值”，玩家会刻意让蛇保持中段长度以扩大控制空间，然后在收尾阶段冲刺多个目标。

3. **障碍 / 迷宫模式**：早期掌机与 HTML5 版本会在原始地图内加入墙体块、移动障碍或陷阱区域，迫使路线规划与躲避难度提升。障碍可能静态阻挡、周期移动或者随着时间解封，强调多目标路径与风险管理。

4. **任务 / 目标节点**：一些现代贪吃蛇衍生如 `Wordalike` 或 `Snake vs Block` 将目标转为“吃到特定序列”或 “触发连锁”，给出先后顺序任务。任务通常是 2~5 个关键节点，失败会有惩罚但不直接死亡，为玩家提供更强目标感。

5. **句子与字符链扩展**：最近的更新版本（例如 `Snakebytes`、`Snakeword`）直接引入拼词系统：吃到的字符会排队成词、短语或公式，完成后获得暴击奖励。与可自由转向的蛇结合后，它为关卡提供了多个对序列的考量维度。

6. **高分结构与奖励**：原版高分策略除了“存活更久”外，还包括“避开自撞区域”“利用障碍巧妙绕行”“在高压阶段塞进连击”。许多平台会设立 combo 补正、X 倍奖励或随机加成以鼓励玩家挑战极限。

上述模式的组合形成了横跨平台的贪吃蛇谱系：经典为基础，限时与障碍增加节奏控制与风险，任务、字符链和奖励层引入明确目标与定向玩法，最终成为更丰富的关卡集合。

## 2. 我们版本的映射与差异化路径

我们是在 Excel 工作簿框架里的模块，已有基础玩法、文字链机制与 HUD 结构。映射逻辑如下：

1. **经典模式照搬**：保留标准蛇体移动、碰壁自撞死亡、分数与长度记录，HUD 用 `SnakeHud` 说明当前状态。地图尺寸（小/中/大）为静态选项，保持原版那种“越长越难”的体验，避免复杂干扰。

2. **限时模式转向“短局冲分”**：原版时间限模式直接套用，但我们强调“目标词 + 时间”双压。时间戳配合文字链进度，让玩家在 60~90 秒内不仅追分，还要完成 `GO`、`HP` 此类任务，提高目标感。

3. **障碍模式升级成“文字链+压力”**：在原版障碍基础上，挑战模式加入静态障碍（合并单元格、冻结窗格）与动态压力障碍（审计扫描线）。吃错字符会产生额外障碍，连动文字链和地图的节奏，既保留路线规划也让文字链失误有后果。

4. **任务 / 目标词层承接**：完全借用文字链系统，将原版任务改为“顺序吃字符”。每个任务关卡给定目标（如 `LOVE`、`EXCEL`、`404`，高阶为 `LOVE YOU`），通过正确完成获得分数、增长奖励。错误的字符会触发惩罚事件（压力等级上升、障碍生成、链尾污染）。

5. **句子与字符链扩展融合**：不做纯粹独立游戏，而是在 `word_chain` 或 `phrase_run` 变体中融入当前文字链：例如 `classic+` 自动识别 `LOVE`、`EXCEL` 等词组成连击；`target_word` 关卡多段句子，完成任一段给予阶段性奖励。故意保留文本可读性与最大 14 个字符可视，避免视觉噪音。

6. **高分 / 奖励结构**：引入 `SnakeTextChain` 的奖励字串、词组倍率与风险奖励（如靠障碍吃词组额外分），等价于原版高分技巧。灯箱遮罩和 HUD 事件提示明确告知玩家何时触发 `chainBonus`、`pressureLevel`，打造现代高分反馈。

7. **不做的项目**：避免加入过多原版之外的玩法如对战模式、大规模地图物件、复杂 meta 系统。重点保持“短局、高频、可复玩的表格式体验”，不让 Snake 变成大型章节工具。

## 3. 正式模式、关卡包与后续架构

### 模式清单（首发 + 后扩）
1. `classic`：标准模式，带自由字符链奖励，适合最纯体验，始终可选地图尺寸。
2. `timed`：60~90 秒冲分系列，首发包含 `easy` 和 `normal` 时间档，后续增加 `ultra`（更短时间）或 `marathon-like`（带字串进度）。
3. `challenge`：目标词与障碍结合，首批 3 套地图（小/中/大）分别对应不同压力模板，后续可引入 `formula_fix` 与 `audit_escape` 关卡。
4. `word_chain`（后续）：以随机词组 + 连击倍率构成，可借助 `SnakeTextChain` 自动识别常见词。
5. `phrase_run`（后续）：句子分段目标 + 章节奖励，适合成就牌。
6. `formula_fix`（后续）：结合 Excel 公式（SUM、VLOOKUP、#REF!）作为关卡目标，与公式世界观直接对齐。

### 关卡包与章节设计
1. `基础训练包`（首发）：4 个关卡，分别引导文字链基础、目标词、障碍与压力，以及短时间目标。每关以目标词 `GO`、`LOVE`、`404`、`HP` 递进，逐步打开 `pressureLevel` 与障碍。
2. `目标词包`（首发）：包含 `LOVE、EXCEL、SUM` 三个任务，围绕顺序吃词组、目标词奖励、错误惩罚演示。
3. `压力地图包`（首发）：三张地图展现不同障碍模板（冻结线、合并区、审计扫描），强调路线规划与压力增长。
4. `高阶句子包`（后续）：如 `LOVE YOU`、`GOOD LUCK`、`VLOOKUP`，每段完成后有阶段奖励，释放更高分与更强障碍。
5. `公式修复包`（后续）：短句形式出现，如 `A1+B2`、`#REF!`，考验目标顺序与干扰字符辨识。

### 地图大小与难度关系
1. `small 12×16`：最轻盈，适合入门、`classic` 低阶、`timed easy`。
2. `medium 15×20`：默认平衡，用于核心 `classic` / `timed normal` / 基础目标包。
3. `large 18×24`：更多直线与压迫空间，用于 `challenge`、高阶任务与句子包，搭配动态障碍。
地图大小还会影响障碍密度、目标词复杂度（大图对应较长词）。

### 障碍与目标词规则
1. **障碍**：静态（合并区、冻结线、保护区）固定布局；动态（审计线、会议占用、筛选墙）按照间隔触发；文字链错误会额外生成一种 `pressureObstacle`，逐步逼近地图中心。
2. **目标词规则**：`targetPlan` 队列推进，`targetProgress` 指明下一个字符。吃到 `@` 视为万能符，`wild` 也能顶替；错误字符导致 `pressureLevel` 升级、尾部污染、 HUD 报警；完成词组提供分数、增长、链条清洗与下一关提示。
3. **奖励判断**：`SnakeTextChain` 会根据链尾匹配识别 `LOVE`、`EXCEL`、`404`，触发倍率与提示，HUD 显示 `lastEventText` 和 `lastEventTone`。

### 首发内容与后续更新位
**首发**：确保 `classic`、`timed`、`challenge` 三种模式上线，配套基础训练包、目标词包与压力地图包；文字链机制贯穿其中，HUD、覆盖层与公式栏完整提示，地图大小与存档记录（分数/长度/目标完成数）同步。
**后续**：重点放在 `phrase_run`、`formula_fix`、`audit_escape` 等高阶关卡，以及每日挑战、成长记录（目标完成档案、词组图鉴）。同时可以为 `challenge` 模式添加 `dynamic pressure` 结构（定期推出新障碍模板）和 `word_chain` 自动词组胜利条件。

## 4. 结语

通过上述方案，贪吃蛇在保持原版基础节奏的同时，将可读的文字链与章节点结合，形成短局、目标、压力一体的成品级模块。下一步可将方案分阶段落地，先把首发模式与关卡包做通，再逐步推出高阶词组与公式拼写挑战，为 Excel 合辑提供具有记忆点的核心玩法。
