# 俄罗斯方块模块正式分析稿

- 日期：2026-04-12
- 模块：`Sheet11`
- 分析框架：`game-studio -> web-game-foundations -> game-ui-frontend`
- 当前结论：俄罗斯方块已经有稳定可玩的基础，但离成品还差“规则专业感、反馈清晰度、持久化记录”和“战斗后回流”

---

## 1. 模块定位

### 1.1 在工作簿合辑里的角色

俄罗斯方块承担的是“稳定心流”和“结构整理”。

它和 Snake 的区别非常明显：

- Snake：冒险、累积风险、随时翻车
- Tetris：控场、整理、维持秩序、持续判断

它在这套工作簿合辑里很重要，因为 Excel 语义和 Tetris 的“整理堆积”天然贴合。

### 1.2 目标玩家感受

玩家进入 Tetris 后应该感受到：

- 方块规则清楚
- 控制响应可靠
- 消行反馈明确
- 越往后越有整理压力
- 失败后想立刻再打一局

如果控制和反馈不稳，Tetris 的价值会迅速下降。

---

## 2. `game-studio` 视角：玩法总判断

### 2.1 当前玩法形态

从现有实现看，Tetris 已经具备完整的基础局能力：

- 有 `marathon / sprint / ultra`
- 有 active / next / hold
- 有基础旋转与墙踢
- 有清行与计分
- 有 ghost
- 有暂停、重开、退出

这说明它已经不是 demo，而是“简化规则的第一版正式玩法”。

### 2.2 当前最明显的问题

Tetris 当前最大的问题不是“功能少”，而是“专业感还不够强”。

主要体现在：

- 规则反馈偏轻
- 模式目标提示不够明确
- 结算层偏薄
- 记录与成长还没接上

换句话说，它已经能玩，但还没形成让人信服的“高质量 Tetris 模块”。

### 2.3 下一阶段目标

Tetris 后续应该从“能打一局”升级到“短局高质量模块”。

重点不是先加更多模式，而是补：

1. 规则反馈
2. 成绩记录
3. 模式目标表达
4. 更完整的操作与设置边界

---

## 3. `web-game-foundations` 视角：架构边界

### 3.1 当前边界状态

Tetris 当前基础边界是健康的：

- `tetrisBoardState.ts` 负责纯状态推进
- `tetrisSelectors.ts` 负责公式栏文案与衍生信息
- `tetrisPieceRegistry.ts` / `tetrisRotation.ts` 独立承担规则数据
- `TetrisSheet.tsx` 负责输入和 tick

这套边界足以支撑继续升级，不需要先推翻。

### 3.2 下一阶段最需要补的边界

#### 第一层：正式成绩边界

Tetris 是天然适合做记录的模块，但现在这层还很薄。

建议至少定义：

- 最高分
- 最多消行
- 最佳 Sprint 时间
- 最佳 Ultra 分数
- 四连消次数

否则 Hub 和状态回流会一直停在占位层。

#### 第二层：模式规则边界

虽然已有 3 个模式，但它们目前更像同一套玩法的不同结束条件。

后续应让模式变成明确的“目标类型”：

- `marathon`：生存与分数
- `sprint`：效率
- `ultra`：限时冲分

并让 HUD、结算页、状态栏都围绕目标变化。

#### 第三层：高级规则边界

现在规则足够跑通，但后面如果继续做深度，就需要提前留层：

- combo
- back-to-back
- t-spin 展示
- lock delay 微调

这些不要直接塞进 UI 组件，而要继续留在纯状态层。

#### 第四层：恢复与设置边界

Tetris 比 Snake 更依赖操作体验，因此必须补正式设置边界：

- ghost 开关
- hold/next 显示开关
- 高对比棋盘
- 动效简化

恢复策略也要更保守，只建议恢复 `marathon`。

### 3.3 推荐的正式模块边界

```text
Tetris Module
├─ board simulation
├─ piece registry
├─ rotation / kick rules
├─ score / goal rules
├─ selectors
├─ ui surfaces
└─ storage / progression
```

## 4. `game-ui-frontend` 视角：界面层分析

### 4.1 当前 UI 方向是否成立

当前 Tetris UI 基本方向是对的：

- 主棋盘居中
- 侧边看板承载 HUD
- overlay 处理 idle / pause / finished / dead
- Excel 壳层整合自然

不需要重做布局范式。

### 4.2 当前 UI 的主要短板

#### 短板 1：目标感不够强

玩家现在能看懂怎么打，但不一定明确“这一局到底在追什么”。

例如：

- `sprint` 应该突出“还差几行”
- `ultra` 应该突出“剩余时间”和“冲分节奏”
- `marathon` 应该突出“等级与堆高压力”

#### 短板 2：结算页不够有回流能力

现在 overlay 能完成状态覆盖，但还不够像正式结算层。

正式版应补：

- 本局成绩
- 是否破纪录
- 本局亮点
- 再来一局 / 返回主页 / 查看记录

#### 短板 3：高水平反馈偏弱

清行、四连消、极速处理这类高光反馈，当前还是偏轻。

Tetris 不需要很吵，但必须“清楚而有分量”。

### 4.3 Tetris 推荐 UI 结构

Tetris 正式版建议固定为 4 个持续面：

1. 主棋盘
2. 左侧或右侧暂存区
3. 未来队列与核心 HUD
4. 模式目标提示与结算覆盖层

不要再加很多说明卡片，避免挤压主棋盘。

## 5. 当前版本能力评估

### 5.1 已经成立的能力

- 基础 Tetris 闭环已经成立
- 纯状态推进边界清楚
- 关键交互已具备
- 与工作簿壳层接通完成
- 模式扩展空间已经预留

### 5.2 当前阻止它继续升级的关键问题

- 成绩记录层不完整
- 模式目标表达不够强
- 结算与复盘层偏薄
- 高阶反馈不足
- 设置与恢复还不是正式模块级

---

## 6. 下一阶段功能包

### 6.1 功能包 A：模式目标强化

这是最优先的一包。

建议目标：

- `marathon`：突出等级与存活压力
- `sprint`：突出剩余行数与完成时间
- `ultra`：突出剩余秒数与冲分效率

只要这一层做好，Tetris 的成品感会立刻上升。

### 6.2 功能包 B：记录与结算

包括：

- 模式最佳成绩
- 本局结算页
- 破纪录提示
- 快速重开与回主页

### 6.3 功能包 C：高阶反馈

建议按从轻到重的顺序补：

1. 单消/双消/三消/四连消明确提示
2. combo / back-to-back 摘要
3. 更清楚的危险堆高提示
4. 更细的结算亮点描述

### 6.4 功能包 D：设置与恢复

包括：

- ghost 开关
- hold/next 显示控制
- 高对比和 reduced motion
- marathon 轻量恢复

---

## 7. 建议的实施顺序

Tetris 不建议先冲复杂 guideline 细节，建议按下面顺序推进：

1. 模式目标强化
2. 记录与结算
3. 高阶反馈
4. 设置与恢复
5. 更深的规则细节

这样能先把“像成品”这件事做出来，再补“像高手向模块”。

---

## 8. 当前正式判断

Tetris 现在已经值得继续投入，但必须克制推进方向。

接下来不该先扩更多功能名词，而应按一句话执行：

**先把 Tetris 做成规则清楚、反馈有分量、记录完整的高质量短局模块，再继续补深度。**
