# 祖玛游戏包设计（建议 Sheet14 / Sheet15）

## 设计前提
- 目标：在当前 Excel 风格游戏中心内，高还原交付经典 Zuma（轨道彩球链 + 炮台插入消除）体验，并预留长期内容扩展位。
- 约束：沿用现有项目 2D 浏览器游戏架构，保持“simulation 与 render 分离、DOM HUD + 游戏主画面并行”的工程边界。
- 输出标准：文档中的模式包可直接拆为实现任务，映射到关卡数据、运行时状态机、UI、存档和测试。

## 1) 设计目标与原版体验支柱

### 设计目标
1. 还原原版祖玛的核心心理循环：瞄准 -> 插入 -> 消除 -> 回缩连锁 -> 危机解除。
2. 维持“短局高压 + 可复盘可重试”的节奏，每局 2~6 分钟为主，失败反馈清晰。
3. 在 Excel 壳层里保持信息可读性，不牺牲原版关键压迫感（终点线、球链推进、颜色管理）。

### 原版体验支柱
- **插入精度支柱**：玩家主要技术来自角度控制、提前量和插入位置判断，不是纯随机运气。
- **颜色管理支柱**：当前球 + 下一球的决策权重高，换弹（swap）是核心高阶动作。
- **回缩连锁支柱**：消除后缺口回缩与二次消除是爽点，必须可见、可预判、可触发连续奖励。
- **终点压迫支柱**：球链接近骷髅口时压力陡增，危险反馈必须持续但不吵闹。
- **关卡地形支柱**：轨道几何、弯道、交叉和遮挡直接改变策略，不只换皮肤。
- **道具节奏支柱**：道具球是节奏修正器，不能喧宾夺主替代基本瞄准能力。

## 2) 为什么适合放进当前游戏中心与建议 Sheet 职责

> 本文中的 Sheet 编号按当前新增三套经典模块的统一接入方案固定为 `Sheet14 / Sheet15`，避免与 Pac-Man、Match-3 的模块页冲突。

### 适配原因
- 与当前 PvZ/Snake/Tetris 一样，祖玛天然适配“短局 + 明确输赢 + 进度存档 + 反复挑战”。
- 规则可拆成明确数据层（轨道、球色池、速度曲线、道具分布），适合项目现有数据驱动风格。
- UI 结构和 Excel 语境兼容：顶部信息条（关卡/分数/进度/危险）、右侧操作区（暂停/倍速/重试）、底部状态栏（连锁/奖励提示）。

### 建议 Sheet 职责
- **Sheet14（祖玛主玩法）**：选关、模式切换、战斗主界面、HUD、局内结算。
- **Sheet15（祖玛图鉴与练习实验）**：球种图鉴、道具说明、轨道练习、命中率训练、回缩连锁教学。

## 3) 核心 loop 与运行规则（原版还原向）

### 3.1 单局核心 loop
1. 关卡加载：生成轨道路径、终点线、初始球链、颜色池和道具规则。
2. 炮台发射：玩家旋转炮台瞄准，发射当前球；可切换下一球。
3. 插入判定：球与链碰撞后插入目标位置，链段重新排序与间距修正。
4. 消除判定：同色连续 >= 3 触发清除，记录得分与反馈。
5. 回缩与连锁：消除产生间隙后发生回缩，若再次形成 >= 3 继续连锁。
6. 危险推进：剩余球链继续朝终点推进；若触终点线则失败。
7. 胜利结算：清空所有在场球链并完成关卡目标。

### 3.2 炮台与输入模型
- 输入动作：`aim`（鼠标/摇杆方向）、`shoot`（左键/确认）、`swapBall`（右键/Tab）、`pause`、`speedToggle`。
- 炮台状态：`currentBall`、`nextBall`、`cooldownMs`、`aimAssist`（仅训练模式）。
- 命中反馈：命中/空枪/擦边需要不同音画反馈，空枪不应“静默失败”。

### 3.3 轨道与彩球链
- 轨道使用离散采样样条（polyline/spline），每颗球以 `distanceAlongTrack` 表示位置。
- 球链结构建议：
  - `chainId`
  - `balls[]`（颜色、是否道具球、状态标记）
  - `headDistance`
  - `speedFactor`
  - `isRewinding`
- 前进逻辑：基础推进速度 + 难度倍率 + 事件修正（减速/加速）。

### 3.4 碰撞、插入与消除
- 发射球与链段碰撞时，计算最近合法插入点（前插/后插），避免穿透与错位。
- 插入后立即进行局部匹配扫描：
  - 从插入点向两侧扩展统计同色连续长度。
  - 若 >= 3 则清除并触发回缩检查。
- 回缩规则：
  - 仅在“前方无推进阻断”且“间隙存在”时回缩。
  - 回缩接合后再次触发消除扫描，允许多段连锁。

### 3.5 道具球体系（首发建议）
- **爆裂球**：命中后半径范围清除。
- **闪电球**：沿轨道方向清除一段连续球。
- **减速球**：全链减速 N 秒。
- **倒退球**：链头后退固定距离，制造喘息窗口。
- **万能球**：可当任意颜色参与三消。
- 规则约束：道具触发率受关卡控制，避免纯靠道具通关。

### 3.6 关卡终点与失败线
- 失败线必须可见（骷髅口/终点门），并在危险距离内给出连续预警。
- 建议危险分层：
  - `safe`：> 35%
  - `warning`：15%~35%
  - `critical`：< 15%
- 进入 `critical` 时，HUD + 音效 + 轨道视觉同步升级。

## 4) 原版感模式、节奏、关卡结构与难度曲线

### 模式结构（首发建议）
- `adventure`（主线征途）
- `challenge`（限定条件挑战）
- `time_attack`（计时冲分）
- `endless`（生存无尽）
- `practice`（训练与教学）

### 难度曲线（建议分 4 段）
1. **入门段（1~12 关）**：2~3 色、单轨、低速，重点学插入与换弹。
2. **进阶段（13~28 关）**：4 色、弯道/遮挡、首次道具球，加入回缩连锁教学。
3. **高压段（29~45 关）**：多轨或分岔、速度波动、连续危机窗口。
4. **终局段（46~60 关）**：高色数、复杂轨道、Boss 规则或多目标通关。

### 原版节奏控制要点
- 每关前 20% 时间给策略建立空间，中段进入稳定高压，末段有一次“收官冲刺”。
- 避免全程线性加速，采用“波段式压力”：推进加速 -> 清除回退 -> 再次加速。

## 5) 正式内容包 / 模式包（8 包）

1. **神庙征途包：经典单轨入门**
   - **定位**：祖玛主线起步，教学瞄准、插入、基础三消、换弹。
   - **原版参考**：Zuma 早期神庙关卡（低色数、单轨、明显弯道）。
   - **核心规则**：2~3 色球池，单条轨道，失败线距离较长；逐关引入“先看下一球再开火”。
   - **建议关卡数**：12 关。
   - **首发必做**：完整插入判定、基础回缩、换弹键、危险分层 HUD。
   - **可延期内容**：入门成就（零空枪、连锁次数达标）。
   - **Excel 对应**：`Sheet14` 默认模式入口；`Sheet15` 提供这 12 关的教学回放与讲解。

2. **丛林弯道包：遮挡与提前量训练**
   - **定位**：强化“盲区弯道”瞄准，提升插入准确性上限。
   - **原版参考**：中期弯道密集地图与遮挡视线关卡。
   - **核心规则**：轨道遮挡增加，要求玩家在球链出弯前预判插入点。
   - **建议关卡数**：8 关。
   - **首发必做**：轨道遮挡渲染、命中点预览（仅训练开关）、弯道命中音画反馈。
   - **可延期内容**：动态遮挡（树叶/机关短时遮挡）。
   - **Excel 对应**：`Sheet15` 新增“弯道命中训练”子页，统计命中率与空枪率。

3. **连锁回缩包：爽点强化章节**
   - **定位**：把祖玛核心爽点“回缩连锁”做成章节主目标。
   - **原版参考**：中后期高连锁收益关。
   - **核心规则**：关卡评分更看重连锁层数和回缩触发次数；部分关卡限制总发射数。
   - **建议关卡数**：8 关。
   - **首发必做**：稳定回缩物理、连锁计数器、连锁分数倍增、连锁特效。
   - **可延期内容**：连锁挑战榜、回缩教学慢放。
   - **Excel 对应**：`Sheet14` HUD 增加 `连锁层数` 与 `本局最高连锁`；结算突出连锁贡献。

4. **机关道具包：道具球策略章节**
   - **定位**：引入爆裂/闪电/减速/倒退/万能球，形成策略选择。
   - **原版参考**：祖玛中后期道具球关与节奏修正关。
   - **核心规则**：道具球按关卡概率刷出；部分关卡要求用指定道具完成目标。
   - **建议关卡数**：10 关。
   - **首发必做**：5 类道具球全链路（生成、碰撞、触发、UI提示、得分）。
   - **可延期内容**：复合道具（减速+连锁加成）与周轮换道具规则。
   - **Excel 对应**：`Sheet15` 图鉴页增加道具球说明与演示；`Sheet14` 关卡简介显示本关道具池。

5. **分岔多轨包：多线危机管理**
   - **定位**：从单链压力升级到多链并行管理，提升全局决策。
   - **原版参考**：双轨/交叉轨道高压关。
   - **核心规则**：同屏 2 条链或分岔链同时推进；玩家需在“即将失守线”与“可连锁线”间切换。
   - **建议关卡数**：8 关。
   - **首发必做**：多链数据结构、目标链优先提示、危险链自动标识。
   - **可延期内容**：链间机关联动（清 A 线触发 B 线减速）。
   - **Excel 对应**：`Sheet14` 顶栏显示 `线路风险排行`；状态栏提示当前最危险链。

6. **Boss 图腾包：阶段战与机制关**
   - **定位**：在祖玛框架内加入阶段式 Boss，不改核心射击逻辑。
   - **原版参考**：Boss 关“多阶段推进 + 机制窗口”。
   - **核心规则**：Boss 关包含 2~3 阶段，每阶段改变速度、轨道或球色池；需达成阶段目标才能推进。
   - **建议关卡数**：6 关（每大章 1 个 Boss）。
   - **首发必做**：阶段状态机、Boss 条件 UI、失败后阶段内快速重开。
   - **可延期内容**：Boss 专属道具与视觉演出强化。
   - **Excel 对应**：`Sheet14` 结算页展示阶段达成表；`Sheet15` 记录 Boss 弱点与建议策略。

7. **计时冲分包：高密度短局**
   - **定位**：满足“再来一把”需求，强调效率和稳定命中。
   - **原版参考**：祖玛计时挑战、街机冲分模式。
   - **核心规则**：固定时长（60/120/180 秒），以得分与连锁效率排名；失败线仍生效。
   - **建议关卡数**：3 个时长档 + 6 张轮换地图。
   - **首发必做**：计时器高优先展示、冲分结算、最佳成绩榜。
   - **可延期内容**：每日种子地图与好友对比。
   - **Excel 对应**：`Sheet14` 模式卡新增 `计时冲分`；状态栏显示“本轮剩余秒数”。

8. **无尽生存包：长期回流内容**
   - **定位**：长期留存模式，提供稳定高压和成长型目标。
   - **原版参考**：祖玛高难生存与无限波段挑战。
   - **核心规则**：波段式提速 + 颜色池扩张 + 道具稀缺；目标是坚持更长波段与更高分。
   - **建议关卡数**：3 张无尽底图（单轨、双轨、交叉）。
   - **首发必做**：波段系统、阶段结算、历史最佳波段记录。
   - **可延期内容**：周赛季词条（例如“本周无减速球”）。
   - **Excel 对应**：`Sheet14` 与 `Sheet15` 联动展示历史最佳、最近失败原因、推荐训练入口。

## 6) HUD、结算、进度记录、练习模式与图鉴/说明页

### HUD（战斗中）
- 必显信息：
  - 关卡编号 / 模式
  - 当前分数 / 连锁层数 / 最高连锁
  - 当前球 + 下一球
  - 轨道推进百分比与危险等级
  - 暂停 / 倍速 / 重开
- Excel 风格要求：
  - 顶部两行内完成主要信息，避免压缩主战场。
  - 关键指标胶囊化显示，危险状态用轻量高对比色标。

### 结算（胜负后）
- **胜利结算**：用时、命中率、最高连锁、道具使用效率、奖励星级。
- **失败结算**：失守轨道、最后 20 秒关键事件、推荐训练项（如“弯道命中”）。
- 必须提供：`重开本局` / `返回选关` / `进入练习`。

### 进度记录
- 存档字段建议：
  - `clearedLevels`
  - `bestScoreByLevel`
  - `bestChainByLevel`
  - `bestAccuracyByLevel`
  - `endlessBestWave`
  - `recentFailReasons[]`
- 展示策略：Sheet15 以“章节进度 + 技术短板”双视图展示。

### 练习模式（Sheet15）
- 模块建议：
  - `弯道命中练习`
  - `回缩连锁练习`
  - `危险线抢救练习`
  - `道具球专项练习`
- 每个练习模块都有可量化指标（命中率、平均连锁、危机化解率）。

### 图鉴 / 说明页（Sheet15）
- 图鉴对象：
  - 普通彩球规则
  - 道具球规则
  - 轨道地形标签（单轨/双轨/交叉/遮挡）
  - Boss 阶段机制摘要
- 说明策略：短句 + 动画示例，不写长段叙事文案。

## 7) 技术实现边界（按现有 2D 浏览器架构）

### 7.1 架构分层（必须）
- **Simulation 层**：纯数据推进，不依赖渲染对象。
  - 建议文件：
    - `src/features/zuma/zumaTypes.ts`
    - `src/features/zuma/zumaBoardState.ts`
    - `src/features/zuma/zumaLevelCatalog.ts`
    - `src/features/zuma/zumaProgressStorage.ts`
- **Render/UI 层**：负责绘制轨道、球链、炮台、特效、HUD。
  - 建议文件：
    - `src/components/zuma/ZumaGameSheet.tsx`
    - `src/components/zuma/ZumaBoard.tsx`
    - `src/components/zuma/ZumaHud.tsx`
    - `src/components/zuma/ZumaResultPanel.tsx`
    - `src/styles/zuma.css`

### 7.2 状态推进建议
- Tick 驱动：`updateZumaBoard(state, input, dtMs)`。
- 事件流建议：
  - `SHOT_FIRED`
  - `BALL_INSERTED`
  - `MATCH_CLEARED`
  - `CHAIN_REWIND`
  - `POWERUP_TRIGGERED`
  - `LEVEL_WON / LEVEL_LOST`
- 事件用于 HUD 文案、音效触发和测试断言，避免从渲染层反推状态。

### 7.3 数据与关卡脚本
- 关卡定义字段建议：
  - `id`
  - `trackId`
  - `colorPool`
  - `baseSpeed`
  - `spawnScript`
  - `powerupPool`
  - `winCondition`
  - `lossCondition`
  - `recommendedPracticeTags`
- 颜色池遵循“渐进扩色”原则，不在前期一次性上满。

### 7.4 输入与可测试性
- 输入映射集中化，避免组件内散落 keydown 逻辑。
- 核心可测点：
  - 插入点稳定性
  - 三消与连锁正确性
  - 回缩触发边界
  - 失败线判定
  - 道具球触发效果
- 测试建议文件：
  - `tests/zumaInsertion.test.ts`
  - `tests/zumaMatchAndRewind.test.ts`
  - `tests/zumaLossCondition.test.ts`
  - `tests/zumaPowerups.test.ts`

### 7.5 性能与调试
- 性能预算：目标 60fps，常规关卡同屏球体 < 220，峰值 < 320。
- 调试开关：
  - 轨道采样点显示
  - 球链索引显示
  - 插入点候选显示
  - 碰撞半径显示
- 禁止把调试逻辑混入正式渲染路径。

## 收口结论
- 祖玛在本项目应以 `Sheet14 主玩法 + Sheet15 图鉴/练习` 的双页结构落地，保持与现有模块一致的信息架构。
- 首发必须优先保障“插入精度、回缩连锁、失败线压迫”三大原版体验，不先追求花哨机制。
- 上述 8 个模式包可直接拆为实现任务，先交付 `神庙征途包 + 连锁回缩包 + 计时冲分包`，再扩展 Boss 与无尽，以最小风险拿到高还原核心体验。
