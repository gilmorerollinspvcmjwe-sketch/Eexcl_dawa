# Zuma 模块 P0/P1/P2 详细扩展方案

- 日期：2026-04-15
- 适用模块：`Sheet14 / Sheet15`
- 目标：基于当前仓库已经落地的祖玛核心运行时，补齐关卡目标、模式层、练习闭环与后半程内容，逐步推进到完整经典祖玛模块
- 约束：继续沿用当前 `状态层 + 关卡目录 + 进度存档 + 双 Sheet UI` 架构，不重写项目外层工作簿系统

---

## 0. 当前仓库基线与设计原则

### 0.1 已有基线

当前 Zuma 已具备以下能力：

1. **核心单局状态层成立**
   - 瞄准发射
   - 插入球链
   - 三消
   - 回缩
   - 回缩后二次连锁
   - 终点失败
   - 危险预警
2. **道具体系已存在**
   - 爆裂 / 闪电 / 减速 / 倒退 / 万能球
3. **内容入口已存在**
   - 神庙征途 12 关
   - 连锁回缩 8 关
   - 计时冲分 18 图
   - 练习配置 4 组
4. **双 Sheet UI 已成型**
   - `Sheet14`：主玩法
   - `Sheet15`：图鉴与训练入口
5. **基础进度与结算已接通**
   - 最佳分
   - 最佳连锁
   - 命中率
   - 计时记录
   - 练习记录

### 0.2 当前真正缺口

当前阻止 Zuma 成为“完整模块”的，不是发射和消除本身，而是以下 5 个缺口：

| 维度 | 当前状态 | 主要缺口 |
| --- | --- | --- |
| 目标系统 | 只有清空与计时胜利 | 限弹、连锁、道具、多目标组合未落地 |
| 内容结构 | 仅前中期 20 关 + 计时图轮换 | 后半主线、高压段、终局段、Boss、无尽缺失 |
| 模式层 | adventure / timed 可玩 | challenge / endless 未成型 |
| 练习闭环 | 有训练配置和训练页 | 入口不通、结算不回流 |
| 长线成长 | 有基础记录 | 无失败原因、短板归因、无尽记录、专项回流 |

### 0.3 三条硬原则

1. **P0 先补真实目标系统，不再发散做新花样**
   先把 spec 里已经写明的限弹、连锁、道具、多目标做实。
2. **内容扩展必须沿用当前目录，不另起平行包**
   先沿神庙 / 连锁 / 计时三条现有线继续长。
3. **UI 继续维持双 Sheet，不额外加第三页**
   主玩法与训练图鉴继续拆在 `Sheet14 / Sheet15`。

## 2. P0：补齐真实主玩法

## 2.1 P0 交付目标

P0 不追求大量新关，而是把当前现有关卡真正做成“能打、能判定、能复盘、能训练”的完整主玩法。

### 本批必须完成

1. **真实目标系统**
   - 清空全链
   - 限定发射数
   - 至少达成若干次连锁
   - 限定道具触发数 / 使用数
   - 多目标组合结算

2. **模式 / 配置真实生效**
   - 切关卡包
   - 切计时时长
   - 切 adventure / timed
   - 切完之后真实重建状态

3. **练习闭环接通**
   - `Sheet15` 的练习卡真正能进入 `ZumaPracticeSheet`
   - `Sheet14` 结算页的“进入练习”要能跳到对应训练项

4. **对局退出路径**
   - HUD 加 `返回选关`
   - 暂停层 / 结算层都有明确回退路径

### 验收标准

- 连锁包 / 挑战关不再是文案目标
- 从主玩法失败后能直达推荐练习
- 从练习页可以返回主玩法
- 切换模式 / 包 / 时长后，开局参数与 UI 一致

## 2.2 P0 关卡与内容补充设计

### A. 神庙征途包（1~12）

这一包不新增数量，重点是把现有 12 关改造成“目标真实生效”的教学主线。

#### 章节结构

1. `1-01 ~ 1-04`：插入基础
   - 目标：清空全链
   - 重点：看当前球 / 下一球
2. `1-05 ~ 1-08`：回缩与连锁
   - 目标：清空全链 + 至少 1~2 次连锁
   - 重点：制造缺口与回缩接链
3. `1-09 ~ 1-12`：道具基础
   - 目标：清空全链 + 道具使用要求
   - 重点：爆裂、减速、倒退、万能球的使用时机

#### 关卡节奏要求

- 每关前 25% 时间给玩家建盘和观察颜色分布
- 中段进入第一次危险预警
- 尾段必须有一次“终点抢救窗口”

### B. 连锁回缩包（13~20）

这一包是 P0 最重要的“目标系统试金石”。

#### 章节结构

1. `13~15`：单次回缩
   - 目标：至少 2 次连锁
2. `16~18`：连续回缩
   - 目标：至少 3~4 次连锁
3. `19~20`：高压连锁
   - 目标：至少 5 次连锁或限定发射数完成

#### 设计原则

- 这批关不扩新轨道，只围绕当前轨道做脚本和目标强化
- 让“连锁次数”第一次成为硬判定条件

### C. 计时冲分包（3 / 5 / 10 分钟）

P0 只收口“显示与真实一致”，不改模式定位。

#### 目标

- 统一 UI 和规则：当前代码继续保留 3 / 5 / 10 分钟
- 明确它不是“撑住不死”模式，而是“在不碰线前提下冲高分”
- 评分结构突出：
  - 总分
  - 最高连锁
  - 命中率
  - 单分钟均分

## 2.3 P0 玩法机制补充设计

### 1. 目标系统模型

在现有 `winCondition / lossCondition / shotLimit / chainTarget / recommendedPracticeTags` 基础上扩展，不新起平行 schema。

#### 建议支持的目标类型

- `clearAll`
- `surviveUntilTime`
- `maxShots`
- `minChainCount`
- `minPowerupUses`
- `scoreThreshold`
- `multiObjective`

#### 目标展示规则

- `Sheet14 setup`：显示完整目标说明
- `Sheet14 in-game HUD`：只显示压缩目标条
- `result panel`：显示目标完成/未完成原因

### 2. 练习闭环

#### 训练标签与结算推荐映射

- `插入精度`
- `前端抢救`
- `换球判断`
- `回缩连锁`
- `道具时机`

#### 流转规则

- 主玩法结算 -> 推荐训练项 -> 跳练习页 -> 练习结算 -> 返回主玩法页

### 3. 模式切换的真实状态同步

切 adventure / timed / 包 / 时长时，必须：

- 重置 `selectedLevelId`
- 重建 `state`
- 更新 `formula/status`
- 清理上局临时危险态和飞行球

## 2.4 P0 UI 展示方案

### Sheet14（主玩法）

#### Setup 态

- 模式 tab
- 包 tab / 章节 tab
- 关卡卡片区
- 本关目标卡
- 轨道说明 / 道具说明摘要
- 开始按钮

#### Play 态

- 顶部 HUD 三组：
  1. **局面**：当前球 / 下一球 / 模式 / 关卡
  2. **压力**：危险等级 / 进度 / 终点警告
  3. **收益**：分数 / 连锁 / 命中率
- 右侧按钮：
  - 暂停
  - 倍速
  - 重开
  - 返回选关

#### Result 态

- 评级
- 关键数据
- 失败原因或目标完成详情
- 推荐训练项
- 出口：
  - 重开本局
  - 返回选关
  - 进入练习

### Sheet15（图鉴与训练）

#### 区块

1. 球色 / 道具图鉴
2. 轨道与危险说明
3. 训练卡片区
4. 训练结果与推荐回主线提示

#### 训练卡片

- 命中率训练
- 前端抢救训练
- 回缩连锁训练
