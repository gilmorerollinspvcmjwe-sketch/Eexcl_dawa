# 奇幻战线 Mass Battle 路线图

目标定位：

- 在 Excel 外壳里实现 `200-400` 活跃单位同屏战斗
- 建立 `80-120` 可解锁兵种的长期收集与编组体系
- 保持“低炫技、高可读、高信息密度”的观察型战场体验
- 让玩家的核心爽点稳定落在：
  - 兵潮推进与阵线挤压
  - 大量单位同屏交火
  - 解锁新兵种、新章节、新编组
  - 像看战争总表一样读懂战局

当前判断：

- 规则核已经集中在 `src/features/fantasy_lane/`，方向正确
- 现阶段更像“可继续扩展的产品级原型”，还不是工业级 runtime
- 后续开发优先级不应放在更炫的特效，而应放在：
  - 大规模同屏稳定性
  - 内容工业化生产
  - 关卡出题能力
  - 长线解锁驱动

---

  - [ ] 一句战术定义

  - [ ] 一个明确弱点
  - [ ] 一个推荐搭档
- [ ] 建立核心 archetype：

---

## 项目原则

后续所有奇幻战线迭代，优先级统一遵循以下原则：

1. 先保证“几百单位同屏也爽”，再谈更炫特效
2. 先保证兵种体系和关卡出题成立，再扩兵种总数
3. 先保证解锁能改变打法，再增加收集数量
4. React 负责 Excel 外壳和信息层，战场本体逐步独立
5. 所有新增内容都要服务于：
   - 大规模战场观感
   - 清晰兵种定位
   - 长线解锁驱动
   - Excel 风格战争观察体验

一句话路线：

> 先把“200 单位同屏也爽”做稳，再把“100+ 兵种解锁也有意义”做厚。
