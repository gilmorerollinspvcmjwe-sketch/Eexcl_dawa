# Excel 风格摸鱼游戏合辑中心设计稿

- 日期：2026-04-11
- 项目：`excel-aim-trainer`
- 设计范围：首页架构、共享系统、五个游戏模块、风格约束、功能总表
- 当前阶段：产品/交互设计确认稿

---

## 1. 设计结论摘要

本项目将从当前的 **Excel 伪装练枪游戏**，升级为一个 **Excel 风格摸鱼游戏合辑中心**。

已经确认的方向如下：

- 总体气质：**半伪装半整活型**
- 产品结构：**共享成长型**
- 游戏关系：**练枪 / 贪吃蛇 / 俄罗斯方块 / 拼豆 / 植物大战僵尸平级**
- 命名包装：**轻包装**
  - 保留原游戏名
  - 主要通过 Excel 外壳、文案、表格布局、状态文案来实现统一气质
- UI 原则：**延续当前项目的 Excel 风格，不改成现代游戏大厅**
- 拼豆定义：**像素拼豆 / 热熔拼豆式治愈创作玩法**

该版本不追求把所有游戏做成同一套路，而是把它们统一到同一个“工作簿宇宙”里：

- 练枪 = 爽感 / 热手 / 爆发
- 贪吃蛇 = 上头 / 短局循环
- 俄罗斯方块 = 整理 / 心流 / 稳定刷分
- 拼豆 = 治愈 / 收集 / 创作
- 植物大战僵尸 = 守线 / 策略 / 推关

---

## 2. 产品目标

### 2.1 一句话定位

一个看起来像 Excel 工作簿、实际上是工位摸鱼游戏合辑的共享成长中心。

### 2.2 核心体验目标

1. 第一眼仍然像当前这个项目，而不是新起一套 UI。
2. 首页从“练枪入口”升级为“合辑总控台”。
3. 五个游戏都有各自鲜明节奏，不互相抢位。
4. 所有游戏共享等级、货币、称号、主题、图鉴、隐藏内容。
5. 用户既能碎片化开一局，也能长期解锁与收集。

### 2.3 不做什么

- 不改成现代网页式卡片大厅
- 不给每个游戏单独做一套完全不同的 UI 语言
- 不把全部游戏强行塞进同一种玩法模板
- 不重包装到失去原游戏辨识度

## 3. 风格总原则

### 3.1 必须保留的视觉骨架

- Excel 标题栏
- 菜单栏 / Ribbon
- 公式栏
- Sheet Tabs
- 状态栏
- 单元格式布局
- 经典绿色为主的工作簿气质

### 3.2 允许强化的表现

- 更完整的报表式分区
- 更明确的行列表头
- 游戏化状态文案
- 条件格式感高亮
- 轻量发光、闪烁、进度变化
- 带一点办公室怪味的系统通知

### 3.3 创意来源

抽象感不主要靠夸张立绘或换皮，而主要来自：

- 文件名
- 公式栏吐槽
- 任务文案
- 系统通知
- 隐藏工作表
- 老板巡检 / 周会 / 审计等事件

## 4. 总体信息架构

项目整体推荐采用如下工作表结构：

| Sheet | 名称 | 作用 | 说明 |
| --- | --- | --- | --- |
| Sheet1 | 游戏中心 | 首页总控台 | 游戏入口、继续游玩、任务、成长、动态 |
| Sheet2 | 游戏区 | 当前游戏容器 | 所有游戏共用壳，仅替换中间主内容 |
| Sheet3 | 统计 | 历史成绩与周报 | 各游戏统计、趋势、最高纪录 |
| Sheet4 | 成长 | 员工等级/称号/货币/解锁 | 长期目标与奖励汇总 |
| Sheet5 | 图鉴 | 拼豆作品、隐藏模板、主题收藏 | 收集向内容中心 |
| Sheet6 | 设置 | 全局设置 | 声音、视觉、皮肤、隐藏模式等 |

当前已有结构：

- `hub`
- `game`
- `stats`
- `settings`

设计上建议继续沿用现有路由/状态切换方式，只把 `hub` 扩展成真正的首页总控表，并补充 `growth` / `collection` 两类工作表。

## 5. 首页（游戏中心）架构

### 5.1 首页职责

首页不是传统“游戏大厅”，而是 Excel 风格的总控工作表，承担以下职责：

1. 进入任意游戏
2. 继续上次游玩
3. 查看今日任务
4. 查看共享成长数据
5. 查看近期解锁 / 通知 / 彩蛋

### 5.2 首页推荐版式

#### 顶部区

- 工作簿标题
- 一句系统文案
- 继续游玩入口
- 今日推荐入口

#### 主体左侧

- 五个游戏构成的 **目录总表**
- 一行一个游戏
- 重点不是大图，而是可读、可扫、可快速启动

#### 主体右侧

- 今日任务
- 等级 / 称号 / 摸鱼币 / 连登

#### 底部区

- 最近动态
- 最近解锁
- 快捷导航（统计 / 成长 / 图鉴 / 设置）

### 5.3 首页目录总表（推荐）

| 游戏 | 状态 | 最佳记录 | 今日次数 | 快速开始 |
| --- | --- | ---: | ---: | --- |
| 练枪 | 建议热手 | 最高分 / 最高连击 | N | 启动 |
| 贪吃蛇 | 数据流通畅 | 最长长度 | N | 启动 |
| 俄罗斯方块 | 堆积待整理 | 最高消行 | N | 启动 |
| 拼豆 | 创作中 | 当前模板完成度 | N | 继续/启动 |
| 植物大战僵尸 | 防线待检查 | 最高关卡 | N | 启动 |

### 5.4 首页交互规则

| 操作 | 行为 |
| --- | --- |
| 单击游戏行 | 选中该游戏，右侧显示摘要，公式栏显示介绍 |
| 双击游戏行 | 直接进入对应游戏 |
| 点击“启动/继续” | 直接进入对应游戏 |
| 悬浮任务 | 公式栏显示奖励说明 |
| 悬浮动态 | 公式栏显示详细提示 |

### 5.5 首页系统文案风格

示例：

- `今日建议：先热身，再摸鱼，再伪装。`
- `工作簿状态：稳定运行中。`
- `系统提示：检测到娱乐效率略高于工作效率。`
- `审计建议：请均衡游玩各模块。`

### 5.6 首页实施级设计（本轮细化）

本轮确认：首页采用 **“顶部强启动条 + 中部游戏目录总表主区 + 右侧任务/成长侧栏 + 底部动态日志”** 的混合结构。

核心原则：

1. 第一屏的主视觉仍然是 **五个游戏目录总表**
2. 但必须同时具备非常强的 **继续游玩 / 今日推荐** 启动能力
3. 首页仍然像 Excel 工作表，不像现代游戏启动器

#### 5.6.1 首页视觉 thesis

像一张会呼吸的工作簿总览表：顶部是工作流启动条，中部是游戏目录与进度信息，右侧是任务与成长，底部是系统日志和快捷跳转。

#### 5.6.2 首页第一屏结构

| 区域 | 位置 | 主要内容 | 设计目标 |
| --- | --- | --- | --- |
| 快速启动条 | 顶部 | 继续游玩、今日推荐、随机摸鱼、当前称号摘要 | 让用户 1 秒内继续开玩 |
| 游戏目录总表 | 中部左侧主区 | 五个游戏平级目录、状态、纪录、启动按钮 | 建立“合辑中心”认知 |
| 任务侧栏 | 中部右上 | 今日任务、进度、奖励、领取状态 | 提供日常目标 |
| 成长侧栏 | 中部右下 | 等级、称号、摸鱼币、连登、本周游玩 | 建立长期留存 |
| 动态日志 | 底部左 | 解锁、成就、系统通知、彩蛋提示 | 提供生命感与整活感 |
| 快捷导航 | 底部右 | 统计、成长、图鉴、设置 | 完整信息架构入口 |

#### 5.6.3 顶部快速启动条

该区域必须强，但不能抢走目录表的 C 位。视觉上建议做成一条高亮工作流行，而不是 banner。

建议内容：

- `继续：拼豆 #014（62%）`
- `推荐：60 秒练枪`
- `随机摸鱼`
- `Lv.12 单元格清道夫`

交互规则：

| 操作 | 行为 |
| --- | --- |
| 点击继续 | 返回上次未完成内容 |
| 点击推荐 | 进入今日推荐玩法 |
| 点击随机摸鱼 | 从可玩游戏中随机进入一个 |
| 悬浮条目 | 公式栏显示更详细说明 |

#### 5.6.4 游戏目录总表（实施级）

目录总表是首页的主视觉核心，不做现代大卡片，继续采用表格式结构。

推荐字段：

| 游戏 | 当前状态 | 最佳记录 | 今日次数 | 启动 |
| --- | --- | ---: | ---: | --- |
| 练枪 | 建议热手 | 最高分 / 最高连击 | N | 启动 |
| 贪吃蛇 | 数据流通畅 | 最长长度 | N | 启动 |
| 俄罗斯方块 | 堆积待整理 | 最高消行 | N | 启动 |
| 拼豆 | 创作中 | 模板 #014 62% | N | 继续 |
| 植物大战僵尸 | 防线待检查 | 最高关卡 | N | 启动 |

实施规则：

| 行为 | 设计要求 |
| --- | --- |
| 单击行 | 高亮该行，右侧同步显示摘要 |
| 双击行 | 直接启动对应游戏 |
| 行内按钮 | 快速开始 / 继续 |
| 行 hover | 公式栏显示一句话玩法说明 |
| 状态高亮 | 使用轻量条件格式色，而不是大面积换皮 |

#### 5.6.5 右侧双栏：任务 + 成长

任务区采用待办清单式表格，不做活动卡片。

任务字段建议：

| 任务 | 进度 | 奖励 | 状态 |
| --- | --- | --- | --- |
| 完成任意 2 局 | 1/2 | EXP + 币 | 进行中 |
| 练枪 10 连击 | 6/10 | 币 + 特效碎片 | 进行中 |
| 完成 1 个拼豆模板 | 0/1 | EXP + 图鉴奖励 | 未开始 |

成长区采用 KPI 表，不做 RPG 面板。

建议字段：

| 指标 | 值 |
| --- | ---: |
| 员工等级 | Lv.12 |
| 当前称号 | 单元格清道夫 |
| 摸鱼币 | 2480 |
| 连续登录 | 6 天 |
| 本周游玩 | 4 / 5 |

#### 5.6.6 底部动态日志与快捷导航

动态日志应像系统日志，而不是消息盒子。

示例：

- 已解锁：夜班蓝主题
- 拼豆新增作品：咖波表情
- 练枪刷新最高连击
- 检测到异常工作表闪烁

快捷导航按钮跳转：

- 统计
- 成长
- 图鉴
- 设置

#### 5.6.7 首页状态机

| 状态 | 表现 |
| --- | --- |
| 新用户 | 顶部推荐练枪/拼豆，任务偏教学 |
| 日常状态 | 继续游玩优先，目录总表稳定显示 |
| 刚打完一局 | 顶部出现短暂结算回流条，任务刷新 |
| 新解锁 | 动态日志高亮，相关模块出现 `NEW` |

#### 5.6.8 首页组件拆分建议

建议把当前 `GameHub` 拆成以下结构：

- `HubQuickResumeRow`
- `HubGameTable`
- `HubTasksPanel`
- `HubProgressPanel`
- `HubActivityLog`

其中：

- `HubGameTable` 是第一优先级
- `HubQuickResumeRow` 是提升启动效率的关键
- `HubActivityLog` 是后续整活和彩蛋扩展的挂点

## 6. 共享成长系统

共享成长是该合辑成立的核心。所有游戏共享长期反馈，但各自产出侧重不同。

### 6.1 共享资源总表

| 系统 | 作用 | 产出来源 | 消耗/用途 |
| --- | --- | --- | --- |
| 员工等级 EXP | 长期成长主线 | 所有游戏 | 提升等级、解锁内容 |
| 摸鱼币 | 通用货币 | 所有游戏、任务、成就 | 解锁主题、皮肤、文案包、隐藏内容 |
| 员工称号 | 身份标签 | 成就/里程碑 | 展示在首页、结算页、成长页 |
| 成就 | 长线目标 | 跨游戏完成条件 | 奖励 EXP / 摸鱼币 / 特殊解锁 |
| 图鉴/收藏 | 收集驱动 | 拼豆、主题、隐藏模板、特殊事件 | 首页展示、收藏页浏览 |
| 主题皮肤 | 视觉长期反馈 | 解锁获得 | 改变工作簿皮肤、边框、状态色 |

### 6.2 等级系统

| 等级元素 | 设计建议 |
| --- | --- |
| 显示位置 | 首页成长区、状态栏、结算页 |
| 增长来源 | 任意游戏完成一局、任务、成就 |
| 节奏 | 高频小增长，阶段性解锁明显奖励 |
| 解锁内容 | 新主题、特殊文案、隐藏工作表、头像/徽章、游戏彩蛋 |

### 6.3 摸鱼币用途

| 用途 | 示例 |
| --- | --- |
| 主题皮肤 | 经典绿、夜班蓝、审计紫、警报红 |

| 特效包 | 练枪反馈、贪吃蛇轨迹、俄罗斯方块消行效果 |
| 文案包 | 系统吐槽语料、事件提示风格 |
| 收藏内容 | 图鉴展示框、拼豆模板包 |

### 6.4 成就体系方向

| 类型 | 示例 |

| 新手成就 | 完成任意一局 |

## 9. 游戏详细功能总表

## 9.1 练枪

### 定位

合辑中的高爆发竞技模块，维持当前项目的王牌地位，但不再是唯一主模式。

### 功能表

| 类别 | 设计 |
| --- | --- |
| 首页状态 | 建议热手 / 近期手感 / 今日局数 |
| 基础模式 | 限时、无尽、生存、爆头、FPS 专项 |

## 9.4 拼豆

### 定位

治愈创作与图鉴收集核心，是合辑里最慢、最安静、最个人化的模块。

### 功能表

| 类别 | 设计 |
| --- | --- |
| 首页状态 | 创作中 / 当前模板进度 / 作品总数 |
| 基础模式 | 模板创作、自由拼图、图鉴浏览 |
| 核心循环 | 选模板 -> 填豆 -> 完成 -> 定型 -> 收藏 |
| 工具系统 | 单点、吸色、填充、对称辅助、撤销重做 |
| 模板来源 | 表情包、办公用品、游戏联动模板、隐藏模板 |
| 创意事件 | 灵感时刻、颜色奖励、隐藏模板出现 |
| 结算指标 | 完成度、耗时、精确度、作品评级 |
| 奖励侧重 | 图鉴、贴纸、主页装饰、个性化主题 |
| 风格气质 | 温和、解压、反差萌 |

### 拼豆创意强化点

- 每个单元格就是天然拼豆位
- 完成后的“热熔定型”流程要有仪式感
- 自由创作内容可反哺首页装饰和徽章系统

### 9.4.1 拼豆实施级设计（本轮细化）

本轮确认：拼豆采用 **“模板模式优先”** 的工作台结构，并支持 **内置模板库 + 用户导入图片转模板**。

其定位不是竞技小游戏，而是：

- 治愈模块
- 图鉴模块
- 个性化模块
- 用户内容模块

#### 9.4.2 默认进入路径

拼豆的默认落点是模板库，而不是自由创作页。

理由：

1. 第一次进入更好上手
2. 最符合“治愈创作型”定位
3. 可以在模板库中自然暴露导入图片与自由创作入口

进入优先级：

| 场景 | 默认落点 |
| --- | --- |
| 第一次进入 | 模板库 |
| 常规进入 | 模板库 |
| 从首页“继续”进入 | 上次未完成作品 |

#### 9.4.3 拼豆总页面结构

拼豆页建议采用三栏式工作台：

| 区域 | 内容 | 作用 |
| --- | --- | --- |
| 左侧栏 | 分类、搜索、筛选、调色板、工具 | 操作导航层 |
| 中央主区 | 拼豆网格 / 模板列表 / 创作画布 | 主工作面 |
| 右侧信息栏 | 模板信息、完成度、颜色统计、奖励预览 | 辅助信息层 |

在不同子页面中，三栏内容切换，但骨架不变。

#### 9.4.4 模板库设计

模板库是拼豆第一页，应继续采用 Excel 表格式列表，而不是图片瀑布流。

推荐字段：

| 模板名 | 分类 | 尺寸 | 难度 | 完成度 | 状态 | 操作 |
| --- | --- | --- | --- | ---: | --- | --- |
| 曜 | 游戏角色 | 32×32 | 高 | 0% | 未开始 | 开始 |
| 咖波表情 | 抽象/表情 | 16×16 | 低 | 62% | 进行中 | 继续 |
| 办公咖啡杯 | 办公室 | 12×12 | 低 | 100% | 已完成 | 查看 |

模板库必须支持：

| 功能 | 说明 |
| --- | --- |
| 分类筛选 | 基础 / 游戏 / 抽象 / 办公室 / 隐藏 |
| 搜索 | 按模板名、标签搜索 |
| 尺寸筛选 | 16×16 / 24×24 / 32×32 / 48×48 |
| 难度筛选 | 低 / 中 / 高 |
| 最近游玩 | 快速回到最近模板 |
| 收藏模板 | 提升回流效率 |

#### 9.4.5 模板分类策略

模板库需要足够丰富，不能只有基础图案。

推荐分层：

##### 基础模板

- 表情
- 动物
- 食物
- 办公用品
- 符号 / 几何

##### 游戏主题模板

- 游戏角色
- 武器图标
- 植物 / 僵尸
- 靶子 / 方块 / 小蛇

##### 抽象模板

- 网梗表情
- 抽象角色
- 迷因头像
- 短视频风格角色形象

##### 隐藏模板

- 故障图标
- 神秘工作表符号
- 系统警报模板
- 特殊徽章

说明：

- 内置模板可以具有“流行文化启发感”，但不应过度依赖具体受保护素材
- 真正满足“想拼任意角色/梗图”的核心能力，来自图片导入转模板

#### 9.4.6 主创作网格

中央工作面是拼豆的核心。

设计要求：

| 层级 | 内容 |
| --- | --- |
| 底层 | 模板参考层（可开关） |
| 中层 | 玩家已放置拼豆 |
| 顶层 | hover 高亮、选区、拖拽反馈 |

视觉要求：

- 每个单元格就是一个拼豆位
- 已放置豆子要有轻微颗粒感
- 未放置区域保持干净
- 支持缩放
- 支持显示错误格 / 未完成格

#### 9.4.7 工具系统

##### 基础工具

- 单点放置
- 拖拽连续填色
- 吸色
- 橡皮
- 撤销
- 重做

##### 进阶工具

- 桶填充
- 对称绘制
- 区域框选
- 自动跳到下一未完成格
- 显示错误格

##### 辅助控制

- 网格缩放
- 显示编号
- 显示颜色统计
- 显示参考模板

#### 9.4.8 导入图片转模板

该能力是拼豆长期可玩性的核心之一。

导入流程建议：

| 步骤 | 内容 |
| --- | --- |
| 1 | 上传图片 |
| 2 | 裁切区域 |
| 3 | 选择模板尺寸 |
| 4 | 选择色数压缩 |
| 5 | 选择像素化风格 |
| 6 | 生成模板 |
| 7 | 进入创作台 |

尺寸建议：

- 16×16
- 24×24
- 32×32
- 48×48

色数建议：

- 8 色
- 16 色
- 24 色
- 32 色

风格建议：

- 标准像素化
- 高对比像素化
- 柔和拼豆风
- 复古游戏机风

生成后支持两种去向：

| 模式 | 说明 |
| --- | --- |
| 参考拼图模式 | 按生成模板一步步填豆 |
| 可编辑模板模式 | 作为自定义模板继续修改 |

#### 9.4.9 完成后的“定型”流程

拼豆必须有明显的完成仪式感。

推荐流程：

1. 完成提示
2. 压板预览
3. 热熔处理中（进度条）
4. 成品展示
5. 收录图鉴
6. 选择后续动作

后续动作包括：

- 存入图鉴
- 设为首页贴纸
- 设为徽章
- 继续下一个模板

#### 9.4.10 拼豆与首页/成长系统联动

| 联动位置 | 说明 |
| --- | --- |
| 首页继续游玩 | 未完成模板可直接回流 |
| 首页动态日志 | 新完成作品、图鉴新增、隐藏模板出现 |
| 成长系统 | 完成作品数、分类完成度解锁称号与主题 |
| 其他游戏 | 通过成就解锁联动模板（靶子、蛇、方块、植物等） |

建议拆成：

- `PerlerPalettePanel`
- `PerlerImportWizard`
- `PerlerFinalizeFlow`
- `PerlerCollectionPanel`

其中优先级建议：

1. `PerlerTemplateTable`
2. `PerlerWorkspace`
3. `PerlerFinalizeFlow`
4. `PerlerImportWizard`

### 9.5.3 章节表（经典章节推进）

#### 章节推进规则

- 每章按照若干常规关 + 旗帜关 + 压力关 + 章节收尾关的结构组织
- 每完成章节解锁对应植物池与新僵尸池
- 关卡之间保留原版那种“逐步扩大策略空间”的感觉，不做过度随机 roguelike 化

### 9.5.4 交互状态表（玩家操作与系统状态）

#### A. 主状态机

| 状态 | 描述 | 可操作 |
| --- | --- | --- |
| 选卡 | 开局前选植物卡组 | 选卡、确认开始 |
| 战斗进行中 | 常规 PvZ 对局 | 选卡、种植、铲除、拾阳光、暂停 |

#### B. 玩家动作表

| 动作 | 输入 | 结果 |
| --- | --- | --- |
| 选择植物卡 | 点击卡槽 | 进入“待种植”状态 |

#### C. 格子判定规则

| 格子状态 | 是否可种 | 说明 |
| --- | --- | --- |
| 空地 | 是 | 直接种植 |

#### D. 战场同步规则

| 系统 | 规则 |
| --- | --- |
| 子弹 | 连续移动，不按格跳动 |

#### E. 大图/复杂战场可读性

虽然本模块是“经典复刻型”，但放进 Excel 壳后仍需加强可读性：

| 区域 | 设计要求 |
| --- | --- |
| 顶部卡槽 | 卡片图标 + 费用 + 冷却覆盖层 |
| 棋盘格 | 行列清晰，但不能压过单位识别 |
| 僵尸警报 | 旗帜波、Boss 波必须有强提示 |
| 状态栏 | 当前阳光、章节、波次、剩余割草机 |

### 9.5.5 模块边界建议

为了后续可实施性，PvZ 模块建议拆成以下几个稳定单元：

| 单元 | 职责 |
| --- | --- |
| PlantRegistry | 管植物定义、费用、冷却、攻击参数 |
| ZombieRegistry | 管僵尸定义、护甲、速度、特殊行为 |
| LevelScript | 每关的波次、章节、出怪脚本 |
| PvZBoardState | 棋盘植物、僵尸、阳光、冷却、割草机状态 |
| PvZRenderer | 棋盘、卡槽、阳光、波次 UI |
| PvZCombatSystem | 攻击、碰撞、子弹、吞噬、爆炸等战斗结算 |

这样后续如果真的落地实现，不会因为单位池太大而把一个文件堆炸。

## 11. 统一事件与彩蛋系统

### 11.1 通用事件

| 事件 | 影响 |
| --- | --- |
| 老板靠近 | 提示增强、界面伪装压力提升 |
| 周会开始 | 某些游戏出现额外干扰 |
| 审计抽查 | 首页动态更新、特定挑战出现 |
| 加班时段 | 夜间主题、节奏更紧张 |
| 异常工作表闪烁 | 提示隐藏内容 |

### 11.2 公式栏吐槽系统

公式栏应成为统一的“文本戏剧层”，示例：

- 练枪：`=你今天的手感有点离谱()`
- 贪吃蛇：`=IF(继续贪, 继续长, 迟早死)`
- 俄罗斯方块：`=压力值^2`
- 拼豆：`=今日心情 + 20%`
- 植物大战僵尸：`=防线完整性 - 你现在的慌张`

### 11.3 隐藏内容方向

- `SheetX`
- `DONOTOPEN`
- `绩效复核`
- `豆豆图鉴`
- `异常日志`

隐藏内容建议通过：

- 成就
- 深夜登录
- 跨游戏条件
- 特定连击 / 连胜 / 完成率

来逐步解锁。

## 13. 推荐落地顺序

### 13.1 内容优先级

1. 首页升级为合辑总控台
2. 练枪转为平级游戏模块
3. 接入贪吃蛇
4. 接入俄罗斯方块
5. 接入拼豆
6. 接入植物大战僵尸
7. 完善成长、图鉴、主题、隐藏内容

### 13.2 原因

- 首页一旦改完，合辑身份就成立
- 练枪可直接复用，是现成内容
- 贪吃蛇 / 俄罗斯方块能快速建立“不是单游戏”的认知
- 拼豆带来节奏反差与图鉴系统价值
- 植物大战僵尸最重，适合作为中后期内容池

## 14. 验收标准（设计层）

如果后续实现符合以下标准，则说明本设计方向成立：

1. 首页一眼看出是 Excel 风格的游戏总控台
2. 五个游戏入口平级且清晰
3. 不同游戏进入后仍明显处于同一套工作簿系统里
4. 玩家能从任何游戏获得共享成长反馈
5. 拼豆与练枪的气质差异明显但不割裂
6. 公式栏、状态栏、系统文案成为统一氛围层
7. 合辑不是“拼盘”，而像一个有生命的工位娱乐系统

## 16. 最终设计判断

该项目最独特的地方，不是“Excel 皮小游戏”，而是：

**把不同摸鱼情绪，统一放进同一个 Excel 工作簿宇宙里。**

它不是纯怀旧，不是纯恶搞，也不是简单合集，而是一套有持续成长、有情绪节奏分工、有办公室幽默感的桌面游戏系统。

如果实现得当，它会具备以下记忆点：

- 外壳极统一
- 内部玩法差异大
- 文案与系统很会整活
- 收藏和成长让人愿意一直开这个“工作簿”
