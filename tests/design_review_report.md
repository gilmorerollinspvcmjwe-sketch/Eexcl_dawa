### 3.2 功能详细分析

#### 3.2.1 核心玩法 ✅

**目标生成系统**：
- 头部：25% 概率，100 分
- 身体：45% 概率，50 分
- 脚部：30% 概率，25 分
- ✅ 概率分配合理，头部作为高风险高回报目标

**连击系统**：
- 5 连击：1.5x
- 10 连击：2.0x
- 20 连击：2.5x
- 30 连击：3.0x
- ✅ 数值设计合理，鼓励连续命中

**难度系统**：
```typescript
DIFFICULTY_SETTINGS = {
  easy: { spawnInterval: [2.5, 3.5], maxTargets: 1 },
  normal: { spawnInterval: [1.5, 2.5], maxTargets: 2 },
  hard: { spawnInterval: [0.8, 1.5], maxTargets: 3 },
  expert: { spawnInterval: [0.5, 1.0], maxTargets: 3 },
}
```
- ✅ 难度梯度设计合理

#### 3.2.2 设置系统问题

**🔴 P0 严重问题**：

1. **`spawnRate` 设置未生效**
   ```typescript
   // SettingsPanel.tsx 中有 UI
   onChange={(e) => onUpdateSettings({ ...settings, spawnRate: parseInt(e.target.value) })}
   
   // 但 useGameLogic.ts 中未使用
   const diffSettings = DIFFICULTY_SETTINGS[settings.difficulty];
   // spawnRate 完全被忽略
   ```

2. **`sensitivity` 灵敏度设置未生效**
   - UI 存在但代码中未使用
   - 应影响鼠标移动速度或点击判定范围

**🟡 P1 中等问题**：

3. **禅模式目标不消失**
   - README 描述"目标不消失"
   - 实际代码中目标仍有 `expiresAt`，会过期
   ```typescript
   // 目标创建时
   const duration = 1000 + settings.targetDuration * 500;
   // 禅模式也应设置更长或永不过期
   ```

4. **`targetDuration` 仅影响基础值**
   - 设置范围 1-10，实际影响 1.5s-6s
   - 范围较窄，玩家感知可能不明显
